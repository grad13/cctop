## 📋 実装要求

### 対象機能
**FUNC-400/401/402/403**: インタラクティブ機能群（4機能）の完全実装

### 実装対象機能詳細

#### 1. FUNC-400: Interactive Selection Mode
- **ファイル選択UI**: ↑↓キーでの選択、Enterで確定
- **表示フィルタリング**: 選択可能ファイルのリスト表示
- **FUNC-300連携**: キー入力管理システムとの統合

#### 2. FUNC-401: Detailed Inspection Mode  
- **統合制御**: FUNC-402（上段）とFUNC-403（下段）の統合制御
- **モード管理**: 詳細表示モードの開始・終了・状態管理
- **キー分散**: FUNC-300からのキー入力を適切に分散

#### 3. FUNC-402: Aggregate Display Module
- **基本情報表示**: FileID・inode・時系列情報の表示
- **集約統計表示**: aggregatesテーブルからの統計データ表示
- **メトリック統計**: First/Last/Max/Avgの4段階統計

#### 4. FUNC-403: History Display Module
- **履歴表示**: イベント履歴の時系列表示
- **ナビゲーション**: 履歴内での上下移動・フォーカス管理
- **ページング**: 大量履歴の分割表示・スクロール制御

## 🎯 実装要件

### 技術要件
1. **FUNC-300統合**: 全キー入力をFUNC-300経由で処理
2. **FUNC-000連携**: aggregates・eventsテーブルからのデータ取得
3. **画面制御**: 80文字幅での適切なレイアウト表示
4. **状態管理**: waiting→selecting→detail状態の適切な遷移

### 品質要件
1. **応答性**: キー入力から画面更新まで100ms以内
2. **データ整合性**: aggregatesテーブルとeventsテーブルの整合性確保
3. **エラーハンドリング**: データ取得失敗時の適切な処理
4. **メモリ効率**: 大量履歴データの効率的な管理
