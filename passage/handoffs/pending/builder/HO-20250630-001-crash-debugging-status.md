## 🎯 **引き継ぎ資料**

### **システムクラッシュ問題の現状**

**問題の本質**:
- `.cctop`ディレクトリ削除時にTerminal.app/Firefox/iTerm2がクラッシュ
- 原因: DatabaseManager.tsリファクタリング（コミット`2b47119`）で導入
- 部分修正済み: コミット`7672cfc`でiTerm2のみ生存可能に

**デバッグ環境**:
```
containers/
├── claude-container-official/    # Apple Container環境（構築済み）
├── run-claude.sh                # デフォルト実行スクリプト
└── protect-documents.sh         # documents保護スクリプト
```

**次の作業**:
1. コンテナ内でClaude Codeを使用してクラッシュ原因を特定
2. `.cctop`ディレクトリ存在チェックとエラーハンドリング強化
3. DatabaseManager初期化時の適切な処理実装

### **Apple Container課題**

**container-runtime-linuxプロセス残留問題**:
- `container stop/kill`後もプロセスが残る
- 解決策: `container system stop` → `container system start`

**マウント制限**:
- 複数の個別マウントは不可（単一ディレクトリのみ）
- 回避策: 全体マウント後、コンテナ内で`protect-documents.sh`実行
