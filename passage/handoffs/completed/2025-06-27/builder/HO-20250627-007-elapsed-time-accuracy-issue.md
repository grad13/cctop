## 🚨 問題概要

ユーザーから報告された問題：
> cctopで起動するじゃないですか  
> その時に、明らかにelapsedの時間が、起動した時よりも前のタイミングなんです  
> つまりcctopと押して3秒しか経ってないのに、elapsedが10秒と表示されてる  
> みたいな。もちろん、.cctopを消去してから操作した結果です

## 📋 現象確認

実際の出力を確認したところ：
```
Modified               Elapsed  File Name                    Event    Lines Blocks Directory
────────────────────────────────────────────────────────────────────────────────
2025-06-27 04:24:58     03:47  cctop                         find         -     16  bin
```

- 現在時刻: 04:24:58
- Elapsed表示: 03:47（3分47秒）
- 問題: 起動して数秒しか経っていないのに、3分47秒と表示されている

## 🔍 調査依頼事項

1. **Elapsed時間の計算ロジック確認**
   - どこで起動時刻を記録しているか
   - どこでelapsed時間を計算しているか
   - MM:SS形式で表示している箇所の特定

2. **タイムスタンプ参照の問題**
   - 起動時刻の取得タイミング
   - ファイルのmodification timeとの混同がないか
   - タイムゾーンの問題がないか

3. **初期化処理の確認**
   - .cctopディレクトリ削除後の初回起動時の挙動
   - 前回実行時の情報を参照していないか

## 💡 考えられる原因

1. **ファイルのmodification timeとの混同**
   - 'find'イベントでファイルのmtimeを使用している可能性
   - 起動時刻ではなく、ファイルの最終更新時刻を基準にしている？

2. **起動時刻の記録タイミング**
   - プロセス起動時刻の記録が遅れている
   - 別のタイムスタンプを誤って参照している

3. **時刻計算のロジックエラー**
   - 現在時刻 - ファイル時刻 を計算している可能性

## 📝 修正要求

1. **正確なelapsed time表示の実装**
   - プロセス起動からの経過時間を正確に表示
   - ファイルの時刻とプロセスの経過時間を混同しない

2. **テスト可能な実装**
   - elapsed timeの計算ロジックを単体テスト可能にする
   - モックできる時刻取得関数の使用
