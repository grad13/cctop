## 🧪 詳細テスト仕様

### **1. 即応性検証（最重要）**

#### **A. 起動時間測定テスト**
```bash
# テスト環境: 通常環境・低スペック環境で実施
time cctop

# 期待値:
# - Viewer起動: < 100ms
# - 初期画面表示: < 100ms
# - 初期化メッセージ表示: < 100ms
```

#### **B. ユーザー体験測定**
- [ ] コマンド実行→視覚的フィードバックまでの時間
- [ ] 「反応が遅い」と感じる時間（主観的評価含む）
- [ ] 10回連続実行での一貫性確認

#### **C. パフォーマンス回帰テスト**
- [ ] メモリ使用量の増加確認
- [ ] CPU使用率のスパイク確認
- [ ] ファイルディスクリプタ使用量確認

### **2. Monitor終了制御検証（気持ちの良さ検証）**

#### **A. シナリオテスト**
```bash
# シナリオ1: Monitor未起動→Viewer起動→Viewer終了
$ ps aux | grep monitor  # Monitor無し確認
$ cctop                  # Viewer起動（Monitor自動起動）
# Ctrl+C                 # Viewer終了
$ ps aux | grep monitor  # Monitor停止確認

# シナリオ2: Standalone Monitor→Viewer起動→Viewer終了
$ cctop-monitor &        # Monitor独立起動
$ cctop                  # Viewer起動
# Ctrl+C                 # Viewer終了
$ ps aux | grep monitor  # Monitor継続確認
```

#### **B. PIDファイル検証**
```json
// .cctop/monitor.pid の形式確認
{
  "pid": 12345,
  "started_by": "viewer",      // "viewer" or "standalone"
  "started_at": 1719456789,
  "config_path": "/path/to/.cctop/config.json"
}
```

#### **C. 異常系テスト**
- [ ] PIDファイル破損時の動作
- [ ] Monitor異常終了時の動作
- [ ] 権限不足時の動作

### **3. プログレッシブローディング検証**

#### **C. 大量データストレステスト**
- [ ] 100,000イベントでの安定動作
- [ ] メモリ使用量の線形増加確認
- [ ] 表示更新の継続性確認
