# HO-20250627-009 一時ファイル規約の制定提案

**作成者**: Builder  
**作成日時**: 2025-06-27 10:30 JST  
**優先度**: Medium  
**推定工期**: 1時間

## 背景

デバッグやテスト時に一時ファイルを作成する際、プロジェクトディレクトリを汚染する問題が発生していました。今回のセッションでユーザーから「一時ファイルに/tmpを使うのは良いアイデアです」との評価をいただきました。

## 提案内容

### 1. 一時ファイル作成規約の制定

以下の規約を`documents/rules/meta/protocols/`に追加することを提案します：

```
P0XX-temporary-file-convention.md
- デバッグ用一時ファイルは必ず/tmpディレクトリに作成
- プロジェクトディレクトリ内への一時ファイル作成を禁止
- 一時ファイルの命名規則（例：/tmp/cctop-debug-*.js）
- セッション終了時の削除推奨
```

### 2. CLAUDE.mdへの追記提案

```markdown
## 一時ファイル規約（Temporary File Convention）

デバッグや検証用の一時ファイルは必ず`/tmp`ディレクトリを使用すること：
- ❌ 禁止: `test-debug.js`, `temp.txt`等をプロジェクト内に作成
- ✅ 推奨: `/tmp/cctop-test-*.js`, `/tmp/debug-*.log`
- 理由: プロジェクトディレクトリのクリーン性維持
```

### 3. Builder/Validatorへの適用

特にBuilder/Validatorエージェントが守るべき規約として強調。

## 期待される効果

1. プロジェクトディレクトリの清潔性維持
2. 誤ってコミットされる一時ファイルの防止
3. デバッグ作業の標準化

## 実装例

今回のセッションでBuilderが使用した例：
```javascript
// デバッグスクリプトを/tmpに作成
cat > /tmp/test-watcher.js << 'EOF'
// デバッグコード
EOF
```
