# Completed Handoffs

完了したすべてのhandoffタスクを日付別・エージェント別に保管します。

## ディレクトリ構造

```
completed/
└── YYYY-MM-DD/
    ├── builder/
    ├── validator/
    ├── clerk/
    ├── inspector/
    └── user/
```

## ワークフロー

1. **タスク完了時**: エージェントが作業を完了
2. **移動**: `in-progress/{agent}/` → `completed/YYYY-MM-DD/{agent}/`
3. **ファイル名**: 元のファイル名を保持

## アーカイブポリシー

- **30日経過後**: `archive/`へ移動を検討
- **90日経過後**: 圧縮保存を検討
- **重要度による保持**: Critical/Highタスクは長期保持

## ファイル整理

### 日付ディレクトリ
- タスク完了日ごとにディレクトリを作成
- 形式: `YYYY-MM-DD`

### エージェント別サブディレクトリ
- 各日付ディレクトリ内でエージェント別に分類
- 検索性と整理の両立

## 注意事項

- 完了タスクのみを配置（in-progressやpendingは不可）
- ファイル内容は変更せず、移動のみ実施
- 定期的なクリーンアップでarchive/へ移動
