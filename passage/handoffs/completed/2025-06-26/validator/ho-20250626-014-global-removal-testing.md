### 1. 基本動作テスト

```bash
# - config.json初期化
# - activity.db作成
# - 監視開始メッセージ表示
```

#### 既存設定での実行
```bash

# 期待結果：

# - 新規作成メッセージなし
# - 通常監視開始
```

### 2. 複数プロジェクト独立性テスト

```bash
# プロジェクトA
mkdir project-a && cd project-a
cctop &
PID_A=$!

# プロジェクトB  
cd ../
mkdir project-b && cd project-b
cctop &
PID_B=$!

# 期待結果：
# - project-a/.cctop/ と project-b/.cctop/ が独立作成
# - 異なるactivity.dbファイル使用
# - 相互に影響しない監視動作
```

### 3. 削除確認テスト

#### 削除されたオプションのエラー確認
```bash
# 削除されたオプションのテスト
cctop --global
# 期待結果：「不明なオプション」エラー

cctop --local  
# 期待結果：「不明なオプション」エラー
```

#### ヘルプメッセージ確認
```bash
cctop --help

# 期待結果：
# - --globalオプションの記述なし
# - --localオプションの記述なし
# - シンプルなヘルプ表示
```

### 4. ファイル生成テスト

#### 正常なファイル構造
```bash
cd test-directory
cctop --daemon

# 期待結果：
test-directory/.cctop/
├── config.json      # 初期設定
├── activity.db      # メインDB
├── activity.db-wal  # SQLite自動生成
├── activity.db-shm  # SQLite自動生成
└── .gitignore       # Git除外設定
```

#### 不要ファイルの生成なし
```bash
# ~/.cctop/ ディレクトリが作成されないことを確認
ls -la ~/.cctop/
# 期待結果：「No such file or directory」
```
