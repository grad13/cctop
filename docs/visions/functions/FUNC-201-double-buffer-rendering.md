# FUNC-201: 二重バッファ描画機能

**作成日**: 2025年6月25日 10:00  
**更新日**: 2025年6月25日 20:45  
**作成者**: Architect Agent  
**Version**: 0.2.0.0  
**関連仕様**: FUNC-200, FUNC-202, FUNC-204  

## 📊 機能概要

CLIアプリケーションの画面描画において、ちらつき（フリッカー）を防止するための二重バッファ方式による描画機能。メモリ上で完全な画面を構築してから一度に画面を更新することで、スムーズな表示更新を実現する。

**ユーザー価値**:
- 画面のちらつきがない快適な表示
- 高頻度更新でも目に優しい
- プロフェッショナルな表示品質

## 🎯 機能境界

### ✅ **実行する**
- メモリ上での画面構築
- 一括画面更新による描画
- カーソル制御によるちらつき防止
- ANSIエスケープシーケンスでの画面制御

### ❌ **実行しない**
- 部分的な差分更新（将来拡張）
- アニメーション効果
- 複雑な描画最適化

## 📋 必要な仕様

### **基本アーキテクチャ**

```
メモリバッファ → 一括描画 → 画面
（画面構築）     （原子的更新）
```

### **実装要件**

#### **1. バッファ管理**
- 現在の画面内容をメモリ上に保持
- 行単位での内容管理
- クリア・追加・更新操作

#### **2. カーソル制御**
- 描画前: カーソル非表示（`\x1b[?25l`）
- 描画後: カーソル再表示（`\x1b[?25h`）
- 位置制御: ANSIエスケープシーケンス使用

#### **3. 描画戦略**
- フル更新: 全行を一括で再描画
- 遅延制御: 16ms間隔での更新制限（60fps）

## 🔧 実装ガイドライン

### **BufferedRendererクラス設計**

1. **バッファ管理**
   - `buffer[]`: 現在の画面内容を行単位で保持
   - `clear()`: バッファクリア
   - `addLine()`: 行追加

2. **描画制御**
   - `render()`: 一括画面更新
   - `hideCursor()/showCursor()`: カーソル制御
   - 遅延レンダリング: 16ms間隔でのデバウンス

3. **ANSIエスケープシーケンス**
   - カーソル移動: `\x1b[{row};{col}H`
   - 行クリア: `\x1b[2K`
   - カーソル表示/非表示: `\x1b[?25h` / `\x1b[?25l`

## 💡 使用シナリオ

### **リアルタイム監視での使用**
```javascript
// ファイル変更を監視し、スムーズに表示更新
monitor.on('change', (events) => {
  renderer.clear();
  events.forEach(event => {
    renderer.addLine(formatEvent(event));
  });
  renderer.render();
});
```

3. **互換性**: 主要ターミナルでの正常動作
