# FUNC-400: Interactive Selection Mode

**作成日**: 2025年6月26日 17:00  
**更新日**: 2025年6月26日 18:40  
**作成者**: Architect Agent  
**Version**: 0.2.3.0  
**関連仕様**: FUNC-202, FUNC-300, FUNC-401  

## 概要

リアルタイム監視表示中にキーボード操作で項目を選択し、詳細情報表示や操作実行を可能にするインタラクティブ選択機能。

## 🎯 機能境界

### FUNC-400の責務
- **選択UI管理**: キーボード操作による単一項目選択インターフェース
- **選択状態管理**: 選択状態の追跡・表示制御
- **色表示制御**: 選択行の背景色・文字色変更
- **遷移制御**: 選択確定時の他機能（FUNC-401）への適切な遷移

### 他機能との境界
- **FUNC-202**: 基本表示はFUNC-202、選択UI層をFUNC-400が追加
- **FUNC-300**: キー入力受信はFUNC-300、選択キー処理はFUNC-400が登録
- **FUNC-401**: FUNC-400で選択→FUNC-401で詳細表示（境界は選択確定時点）
- **FUNC-207**: 選択色設定はFUNC-207、色適用制御はFUNC-400
- **FUNC-201**: 表示更新はFUNC-201、選択状態の表示制御はFUNC-400

### 責務外の除外事項
- **ファイル操作**: ファイルのコピー・移動・削除等は対象外
- **エディタ統合**: 外部エディタ起動は対象外
- **マルチ選択**: 複数項目の同時選択は対象外
- **フィルタリング**: 選択対象の絞り込み機能は対象外

## 📋 技術仕様

### アーキテクチャ設計

#### 状態遷移フロー
```
待機状態 ──|上下キー|──→ 選択状態 ──|Enterキー|──→ 詳細状態
(FUNC-202)                (FUNC-400)              (FUNC-401)
    ▲                         │                      │
    │                      |Escキー|                 │
    │                         │                  |Escキー|
    └─────────────────────────┘                      │
           選択終了                                   ▽
                                                 待機状態へ戻る
```

#### 責務分担
- **待機状態 (FUNC-202)**: ファイル監視データの基本表示
- **選択状態 (FUNC-400)**: 選択UI（キー処理登録 + 行の色変更）
- **詳細状態 (FUNC-401)**: 選択ファイルの詳細情報表示
- **FUNC-300**: システム全体のキー入力管理・分散
- **FUNC-207**: 選択色のテーマ管理

### キーボード入力処理仕様
```javascript
// FUNC-300連携によるキー処理
FUNC300_KeyInputManager
├── Raw Input Reception    // 生キー入力受信
├── State-based Dispatch   // 状態に応じた処理分散
└── PIL002_KeyHandler      // FUNC-400への処理委譲
    ├── ArrowUp/Down       // 選択移動
    ├── Enter              // 選択確定
    └── Escape             // 選択終了
```

### 状態管理仕様
```javascript
SelectionState {
  mode: 'waiting' | 'selecting',  // 待機状態 | 選択状態
  currentIndex: number,           // 選択中の項目インデックス
  selectedFile: string | null,    // 選択されたファイル名
  colorTheme: SelectionColorTheme // 選択色テーマ
}
```

## 機能仕様

### 基本動作

#### 待機状態 → 選択状態
- **トリガー**: 待機状態で上下キー（↑↓）入力
- **状態変化**: waiting → selecting
- **初期選択**: 最上位の表示項目を自動選択
- **表示変化**: 該当行の色(背景色・文字色)が変わる

#### 選択状態での操作
- **↑キー**: 前の項目を選択（行の色変更）
- **↓キー**: 次の項目を選択（行の色変更）
- **Enter**: 詳細状態へ移行（FUNC-401）
- **Esc**: 待機状態へ戻る

### 表示仕様

#### 選択状態の視覚的表現
- **選択行**: 背景色・文字色の変更による強調表示
- **非選択行**: 通常の表示色
- **色設定**: FUNC-207テーマシステムとの統合

## 技術仕様

### 依存関係
- **FUNC-300**: キー入力管理（必須）- 選択キーの受信・分散
- **FUNC-201**: 二重バッファ描画（選択状態の滑らかな更新）
- **FUNC-202**: CLI表示統合（表示基盤）
- **FUNC-401**: 詳細検査モード（選択確定時の遷移先）

### FUNC-300連携要件
- **キーハンドラー登録**: 選択関連キー（↑↓Enter Esc）をFUNC-300に登録
- **状態通知**: 選択状態の開始・終了をFUNC-300に通知
- **優先順位設定**: 選択状態時の他機能キー処理を無効化

### 状態管理
FUNC-400の状態管理インターフェース要件：
- **システム状態**: waiting/selectingの状態追跡
- **選択位置**: 現在選択中の項目インデックス管理
- **色テーマ**: 選択時の背景色・文字色の管理

## 🎨 設定項目

### config.json設定（FUNC-101統合）
```json
{
  "selection": {
    "enabled": true,
    "colors": {
      "background": "#0066cc",
      "foreground": "#ffffff",
      "theme": "default"
    }
  }
}
```

### 主要設定
- **enabled**: 選択モード機能の有効/無効
- **colors.background**: 選択行の背景色（16進数色 or プリセット色名）
- **colors.foreground**: 選択行の文字色（16進数色 or プリセット色名）
- **colors.theme**: FUNC-207テーマシステムとの連携設定

### FUNC-207テーマ連携
選択色はFUNC-207テーマシステムと統合管理：
```json
// .cctop/themes/default.json
{
  "display": {
    "selection": {
      "background": "#0066cc",
      "foreground": "#ffffff"
    }
  }
}
```

## 🚀 実装仕様

### 実装要件
1. **SelectionManager クラス**
   - 選択状態の管理・更新
   - キーボード入力の処理分散
   - 他レイヤとの連携インターフェース

2. **FUNC300KeyConnector モジュール**
   - FUNC-300へのキーハンドラー登録
   - 選択関連キー（↑↓Enter Esc）の処理callback実装
   - 状態変更時のFUNC-300への通知

3. **SelectionRenderer クラス**
   - 選択行の色変更制御
   - FUNC-201二重バッファとの統合
   - FUNC-207テーマシステムとの連携
   - 選択状態の視覚的フィードバック

4. **ColorThemeManager モジュール**
   - 選択色のテーマ管理
   - config.jsonからの色設定読み込み
   - 背景色・文字色の動的適用

## 制限事項

### 技術的制限
- **ターミナル依存**: ターミナルのキーボード処理能力に依存
- **表示領域**: ターミナルサイズによる選択可能項目数制限
- **並行処理**: 大量ファイル変更時の選択応答性低下

### 動作制限
- **単一選択**: 1項目のみ選択可能
- **色テーマ依存**: ターミナルの色表示機能に依存
- **更新頻度**: 高頻度更新時の選択状態リセット
