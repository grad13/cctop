## 📋 概要

Dominantsは、プロジェクトの最高位原則を格納するディレクトリです。ここに配置される原則は、すべての活動の基盤となる不変の公理として扱われます。

## 🔺 Dominantとは

### 定義
- **最高位の規則・原則**: プロジェクト全体の方向性を決定する根本原則
- **不変の公理**: 検証不要で、変更されることのない基本原理
- **メタレベル最高**: すべての改善活動の上位に位置する概念

### 特徴
1. **変更禁止**: 一度確立されたDominant原則は変更されない
2. **常時適用**: 状況に関わらず常に適用される
3. **優先順位最高**: 他のすべてのルール・仮説より優先される

## 📁 現在のDominant原則

### DDD0: プロジェクト改善の階層性原則
すべての作業・改善活動には階層があり、より高次の改善が常に優先される。

**階層**:
1. **[Dominant]** システム全体の改善メカニズム
2. **[Meta]** 方法論・プロセスの改善  
3. **[Concrete]** 個別実装・問題解決

**詳細**: `ddd0-hierarchical-improvement-principle.md`参照

### DDD1: Agent役割必須システム
全てのClaude Code Assistantは必ず1つの役割を持つ。例外は一切認めない。

**定義された役割**（5エージェント体制）:
- **Builder**: 機能実装・コード作成・技術実装
- **Validator**: 品質検証・テスト・デプロイ・品質保証
- **Architect**: システム設計・仕様策定・技術方針決定
- **Clerk**: 文書管理・プロセス策定・CLAUDE.md管理
- **Inspector**: 統計監視・データ分析・システム運用監視

**詳細**: `ddd1-agent-role-mandatory-system.md`参照

### DDD2: 階層メモリメンテナンス原則
プロジェクト内の全文書は、L0→L1→L2→L3の階層構造で管理され、アクセス頻度と重要性に応じて自動的に移行される。

**階層定義**:
- **L0**: 現在のセッション記憶（揮発性）
- **L1**: documents/agents/status/{agent}.md（高頻度更新）
- **L2**: documents/records/（中頻度参照）
- **L3**: documents/archives/（長期保存）

**詳細**: `ddd2-hierarchy-memory-maintenance.md`参照

## 🚫 運用ルール

### 新規Dominant原則の追加条件
1. **普遍性**: あらゆる状況で適用可能
2. **根本性**: 他の原則の基盤となる
3. **自明性**: 説明不要なほど明確

### 禁止事項
- 既存Dominant原則の修正・削除
- 状況依存的な内容の追加
- 実装詳細の記載
