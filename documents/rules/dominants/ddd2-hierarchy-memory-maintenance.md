## 🔺 Dominant宣言

## 📋 基本原則

### 階層メモリメンテナンス原則
**プロジェクト内の全文書は、CPUキャッシュアナロジーに基づくL0→L1→L2→L3の階層構造で管理され、アクセス頻度と重要性に応じて定期的に移行される。**

#### 移行プロトコル体系
1. **L0→L1**: 日次・セッション終了時（エージェント自己管理）
2. **L1→L2**: 週次・月曜日（P044 L1→L2移行プロトコル準拠）
3. **L2→L3**: 週次・月曜日（P043 L2→L3アーカイブ移行プロトコル準拠）

### 階層定義（絶対的基準）
1. **L0（揮発性記憶）**: 現在のセッション内作業記憶
2. **L1（高速キャッシュ）**: documents/agents/status/{agent}.md - リアルタイム更新領域
3. **L2（中速キャッシュ）**: documents/records/ - 完了作業の詳細記録領域  
4. **L3（長期保存）**: documents/archives/YYYY/MMDD-MMDD/ - 週次アーカイブ（P043 L2→L3アーカイブ移行プロトコル準拠）

## ⚠️ 絶対的ルール

### 1. 階層移行の強制実行
- **日次**: L0→L1移行（セッション終了前に必須・status記録）
- **週次**: L1→L2移行（P044準拠・月曜日実施・完了作業の移行）
- **週次**: L2→L3移行（P043準拠・月曜日実施・長期保存への移行）

### 2. 配置基準の絶対性
文書の配置は以下のフローチャートに厳密に従う：

```
記録すべき内容がある
    ↓
バグに関する内容か？
    ├─ Yes → 未解決: records/bugs/ （フラット構造）
    │        解決済: documents/archives/bugs/ へ即座移動
    │
    └─ No → 分析・設計・実装の記録か？
             ├─ Yes → records/reports/
             │        （1レポート1トピック厳守）
             │
             └─ No → インシデントか？
                      ├─ Yes → records/incidents/
                      └─ No → Clerkに相談
```

### 3. 1レポート1トピック原則
**records/配下の全サブディレクトリで絶対遵守**：
- 複数トピックの混在は厳禁
- 既存の複合レポートは歴史的記録として保持
- 新規作成時は必ず単一トピックに焦点

## ✅ 必須実行事項

### 1. エージェント別責務
**全エージェントは以下を厳守**：

#### 共通責務（5エージェント体制）
- **L0→L1移行**: セッション終了前にdocuments/agents/status/{agent}.mdへ記録
- **L1→L2移行**: 週次月曜日にP044準拠で実施（自己管理）
- **role確認**: status更新後はdocuments/agents/roles/{agent}.mdを読み返す

#### Builder/Validator/Architect/Inspector
- 自身のstatus管理（進捗記録）
- 完了作業のL2移行（P044準拠）
- 1レポート1トピック厳守

#### Clerk Agent（特別責務）
- 階層メンテナンス主管
- 配置基準の最終判断権限
- P044/P043実施状況の監督

## 🔒 技術的実装

### 階層間の関係性
- **L0→L1**: 現在進行中作業の即座記録（情報損失防止）
- **L1→L2**: 完了作業の構造化・詳細化（知識の体系化・P044準拠）
- **L2→L3**: 長期保存への移行（アクセス効率の維持・P043準拠）

### records/配下の構造（L2）とarchives/構造（L3）
```
documents/records/ (L2)
├── bugs/        # バグ記録（フラット構造、解決後即archive）
├── reports/     # 分析・設計・実装レポート（P043でarchive移行）
└── incidents/   # インシデント記録

documents/archives/ (L3・P043準拠)
├── 2025/0616-0622/  # 週次アーカイブ（月曜開始）
├── 2025/0609-0615/  # キーワード付与済み
└── 2024/1216-1222/  # patterns検索対応
```

## 📊 違反時の対応

### 違反レベルと対応
1. **配置ミス**: 即座に正しい場所へ移動
2. **1トピック違反**: レポート分割を強制
3. **メンテナンス遅延**: 遅延理由の記録と即座実施

### エスカレーション
- 1回目: 警告と是正指導
- 2回目: インシデント記録
- 3回目: プロセス全体の見直し
