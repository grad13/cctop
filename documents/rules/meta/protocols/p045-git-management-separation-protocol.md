# P045: Git管理分離プロトコル

**作成日**: 2025年6月24日  
**作成者**: Clerk Agent  
**上位規則**: DDD1, PLAN-20250624-005  
**適用範囲**: 全エージェント・全git操作

## 基本原則

### 🎯 **完全分離体制**
プロジェクトは**2つの独立したgitリポジトリ**で管理され、相互に干渉しない物理的境界を維持する。

### 📁 **責任分離定義**
```
06-cctop/           # 親gitリポジトリ
├── .git/           # プロジェクト管理専用git
├── documents/      # 全プロジェクト文書
├── passage/        # エージェント協調システム  
├── code/           # コード関連統合ディレクトリ
│   ├── main/       # メインコードベース（旧cctop/）
│   ├── worktrees/  # 並行開発環境
│   ├── containers/ # コンテナ・デバッグ環境
│   └── releases/   # リリースパッケージ管理
├── CLAUDE.md       # プロジェクト指針・ルール
└── VERSIONs/       # プロジェクト履歴

code/main/          # 子gitリポジトリ（旧cctop/）
├── .git/           # ツール開発専用git
├── src/            # ソースコード
├── test/           # テストスイート
├── package.json    # Node.js設定
└── README.md       # ツール固有文書
```

## 判定ルール

### ⚡ **機械的判定基準**
ファイルパスで即座に判定：

#### **親git対象**
- `documents/` 配下すべて
- `passage/` 配下すべて
- `code/worktrees/` 配下すべて
- `code/containers/` 配下すべて
- `code/releases/` 配下すべて
- `CLAUDE.md`
- `VERSIONs/` 配下すべて
- `.gitignore` (親gitのみ)

#### **子git対象**  
- `code/main/src/` 配下すべて
- `code/main/test/` 配下すべて
- `code/main/package.json`, `code/main/package-lock.json`
- `code/main/jest.config.js`, `code/main/vitest.config.js`
- `code/main/scripts/` 配下すべて
- `code/main/README.md`, `code/main/docs/`

#### **境界ケース処理**
- **迷った場合**: ファイルの物理的配置場所で判定
- **新規ディレクトリ**: 親ディレクトリの管轄に従う
- **設定ファイル**: 影響範囲で判定（全体→親git、ツール→子git）

## Agent別責任

### 🔧 **Builder/Validator**
- **主要git**: 子git（コード・テスト作業）
- **限定的親git**: ドキュメント更新時のみ
- **必須確認**: 作業開始時にディレクトリ・git確認

### 📋 **Architect/Clerk/Inspector**  
- **主要git**: 親git（文書・管理作業）
- **限定的子git**: コード確認時の読み取りのみ
- **必須確認**: git操作前にP045判定ルール適用

## 作業手順

### 🚀 **作業開始時**
1. **ディレクトリ確認**: `pwd`で現在位置確認
2. **対象ファイル確認**: 編集・作成するファイルのパス確認
3. **P045判定**: 上記判定ルールで適切なgit確認
4. **git移動**: 必要に応じて正しいディレクトリに移動

### ✅ **コミット前**
1. **CHK006実行**: Git操作前確認チェックリスト
2. **ファイル確認**: `git status`で対象ファイルが適切か確認
3. **境界確認**: 他git管理ファイルが混入していないか確認
4. **コミット実行**: 確認完了後にコミット

### 🔄 **エラー時対応**
1. **境界違反検出**: 即座に作業停止・状況分析
2. **ファイル移動**: `git rm --cached`で誤追加ファイル除外
3. **適切なgit**: 正しいリポジトリで再度追加・コミット

## 物理的保護

### 🛡️ **.gitignore強制**
```gitignore
# 親git (.gitignore)
code/main/
.DS_Store
VERSIONs
VAULTs

# 子git (code/main/.gitignore) 
../../documents/
../../passage/
../../CLAUDE.md
../../VERSIONs/
../worktrees/
../containers/
../releases/
node_modules/
*.log
coverage/
```

### 🚫 **強制的境界防止**
- **親git**: code/main/への物理的アクセス不可
- **子git**: 親git管理領域（documents/, passage/, worktrees/, containers/, releases/）への物理的アクセス不可
- **自動保護**: .gitignoreにより意図しない追加を完全ブロック

## 例外処理

### ⚠️ **境界横断が必要な場合**
1. **読み取り参照**: 他gitファイルの確認は許可（読み取り専用）
2. **情報共有**: handoffsシステム経由での情報交換
3. **緊急時**: ユーザー明示指示時のみ例外的操作許可

### 🔧 **特殊ケース**
- **VERSIONs/cctop関連**: 親gitで管理（プロジェクト履歴）
- **code/main/README.md**: 子gitで管理（ツール説明）

- **設定ファイル**: 影響範囲に応じた判定
