## 🎯 概要

ユーザー説明の要約・確認を体系的に行うことで、コミュニケーション齟齬を完全に防ぎ、「聞いていない」指摘をゼロにする。

## 📋 理解確認テンプレート

### 必須使用場面
- 新しい問題の説明を受けた時
- 複雑な要求を受けた時
- 不明瞭な指示を受けた時

### テンプレート
```markdown
## 理解確認

私の理解を確認させてください：

**問題の本質**：
- [ユーザーの言葉をそのまま引用]
- [自分の言葉での要約]

**重要なポイント**：
1. [ポイント1]
2. [ポイント2]
3. [ポイント3]

**確認事項**：
- [不明な点や曖昧な表現]

これで正しいでしょうか？
```

## 🚨 トリガーワード検出

以下の言葉が出たら**即座に理解の見直し**：

### 危険シグナル
- 「再三」「何度も」「さっき言った」
- 「違う」「そうじゃない」
- 「聞いてる？」「真面目に」

### 対応手順
1. 即座に作業停止
2. 理解確認テンプレートを使用
3. ユーザーの承認を得てから作業再開

## ✅ 実践例

### 良い例
```
ユーザー：「ガチャガチャ移動すると動く」

私：「理解確認させてください：
- 最初の操作では動かない
- 複数回キー操作（ガチャガチャ）すると動き始める
- その後は継続的に動作する
ということですね？」

ユーザー：「その通りです」
```

### 悪い例
```
ユーザー：「ガチャガチャ移動すると動く」

私：「なるほど、currentIslandIndexの問題ですね」
（理解確認なし、自分の解釈で進める）
```
