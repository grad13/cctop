## 🎯 概要

エージェントの作業特性に応じた記録システムにより、実用的で持続可能な記録プロトコルを確立する。技術的強制力と自己監視システムの組み合わせで、記録忘れを根絶し、作業の可視性・トレーサビリティを向上させる。

## 💻 Coder Agent記録システム（強制型）

### 基本方式
```
指示受信 → 「作業開始」記録 → コード作業 → 「重要進展」記録 → 「完了記録」→ コミット
```

### 技術的強制メカニズム
**pre-commitフック実装**:
```bash
#!/bin/sh
# 30分以内にstatus更新がない場合コミットブロック
if [ $DIFF -gt 1800 ]; then
    echo "🚫 COMMIT BLOCKED: Coder status未更新"
    exit 1
fi
```

### 記録タイミング
1. **開始記録**: 指示受信→即座に作業概要記録（必須）
2. **進展記録**: 重要な進展時（エラー解決、機能完成、方針変更）
3. **バグ記録**: バグ発見・再現・解決時は`documents/bugs/`への記録（義務）
4. **完了記録**: コミット前に必ず結果記録（pre-commitで強制）

## 📊 Inspector Agent記録システム（自己監視型）

### 基本方式
```
surveillance/変更検知 → 作業フラグON → 5分後チェック → status未更新 → 警告・統計記録
```

### 自己監視メカニズム
**surveillance/self-watch.js実装**:
```javascript
// surveillance/ディレクトリ変更を検知
chokidar.watch('surveillance/').on('change', (path) => {
    monitorWorkDetected = true;
    setTimeout(checkStatusUpdate, 5 * 60 * 1000);
});
```

### 記録タイミング
1. **開始記録**: 指示受信→作業概要記録（推奨）
2. **発見記録**: 重要な発見、問題、数値変化時
3. **完了記録**: 作業完了時の結果・統計

## 🐛 Coderバグ記録義務化システム

### 対象範囲
**Coderが発見・対応するすべてのバグ**:
- 実装中のバグ発見
- ユーザー報告バグの調査・修正
- 既存コードの問題発見
- 表示・動作・パフォーマンス問題

### 記録フォーマット
```markdown
# [YYYYMMDD] バグタイトル

**発見日時**: YYYY年MM月DD日 HH:MM  
**重要度**: 高/中/低  
**ステータス**: 発見/調査中/修正中/解決済み

## 📋 現象
- 何が起きているか
- 再現手順
- 影響範囲・頻度

## 🔍 原因分析
- 根本原因
- 関連コード箇所
- なぜ発生したか

## 🔧 修正内容
- 実装した修正
- 変更ファイル一覧
- テスト実施内容

## ✅ 確認事項
- 修正効果確認
- 副作用チェック
- 同種問題の予防策
```

### 運用ルール
1. **即座記録**: バグ発見から30分以内
2. **1ファイル1バグ**: 1つのバグにつき必ず1つの専用ファイル（絶対制約）
3. **継続更新**: 調査・修正の進展を随時更新
4. **完了確認**: 解決後は必ずステータス更新

## ⚠️ 緊急回避手順

### pre-commitフック無効化（Coder緊急時）
```bash
# 一時的無効化
git commit --no-verify -m "emergency: [理由]"

# フック削除（最終手段）
rm .git/hooks/pre-commit
```

### self-watch停止（Inspector緊急時）
```bash
# プロセス停止
pkill -f self-watch.js

# 自動起動無効化
npm run monitor  # safeモードではなく通常モード
```
