## 1. 概要

ファイル・ディレクトリの命名規則違反をゼロにするための強制メカニズム。P006の命名規則を確実に適用するためのプロセスとチェックリストを定義。

## 2. 命名規則違反の典型例

### 畳語違反
- ❌ `specifications/taskgrid/taskgrid-specification.md`
- ❌ `roadmaps/authentication/authentication-roadmap.md`
- ✅ `specifications/taskgrid/design.md`
- ✅ `roadmaps/authentication/roadmap.md`

### ディレクトリ名重複
- ❌ `specifications/design-specification.md`
- ❌ `protocols/protocols-guide.md`
- ✅ `specifications/design.md`
- ✅ `protocols/guide.md`

## 3. ファイル操作前チェックリスト（必須）

### 新規作成・リネーム前の確認事項
```
□ 英語のみ使用しているか？
□ kebab-case形式になっているか？
□ ディレクトリ名との重複はないか？
□ 同じ単語の繰り返しはないか？
□ 意味は明確で簡潔か？
```

### チェック実行タイミング
1. **新規ファイル作成時**
2. **ファイルリネーム時**
3. **ディレクトリ作成時**
4. **大量ファイル操作時**（特に注意）

## 4. 強制メカニズム

### Phase 1: 手動チェックリスト（現在）
- 全エージェントがファイル操作前に必ず実行
- チェックリスト未実行での操作は違反

### Phase 2: 半自動検証（計画中）
```bash
# 検証スクリプト例
check_filename() {
  filename=$1
  dirname=$(dirname $filename)
  basename=$(basename $filename)
  
  # 畳語チェック
  for word in $(echo $dirname | tr '/' ' '); do
    if [[ $basename == *$word* ]]; then
      echo "WARNING: '$word' が重複しています"
    fi
  done
}
```

### Phase 3: 自動検証（将来）
- pre-commitフックでの自動チェック
- リアルタイム検証

## 5. 違反検出時の対応

### 即時対応
1. **作業停止**: 違反を発見したら即座に修正
2. **関連更新**: リンクや参照の更新
3. **記録**: 違反内容と修正をstatus/{agent}.mdに記録

### 予防対応
1. **パターン分析**: よくある違反パターンの把握
2. **プロセス改善**: チェックリストの更新
3. **教育**: 違反事例の共有

## 6. 成功指標

### 測定項目
- 命名規則違反発生数: 0件/週
- チェックリスト実行率: 100%
- 違反検出から修正までの時間: 5分以内

### 期待効果（H008検証結果）
- **即時**: 違反の即座発見と修正
- **短期**: 違反発生率の大幅削減（1週間で0件達成）
- **長期**: 命名規則の完全定着

## 7. よくある認知バイアスと対策

### バイアス1: 「自然な名前」の錯覚
- **例**: `authentication-roadmap.md`が自然に見える
- **対策**: ディレクトリ名を常に意識

### バイアス2: 大量処理時の注意力低下
- **例**: 複数ファイル処理時のチェック漏れ
- **対策**: 1ファイルごとにチェックリスト実行

### バイアス3: 慣れによる省略
- **例**: 「いつものパターンだから大丈夫」
- **対策**: チェックリストの機械的実行
