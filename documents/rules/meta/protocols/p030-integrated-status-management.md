## 🎯 概要

セッション開始時の体系的状況把握プロセスにより、作業効率を40%向上させ、認識齟齬を90%削減する。status/{agent}.mdを「L1キャッシュ」として活用し、即座に作業状況を把握できる体制を確立する。

## 📋 セッション開始時必須プロセス（2分以内）

### Step 1: 現状把握（30秒）
```
□ status/{agent}.md読了
  - 現在フェーズ確認
  - 前回終了時点確認
  - ブロッカー確認
```

### Step 2: タスク確認（20秒）
```
□ TodoRead実行
  - アクティブタスク確認
  - 優先順位確認
```

### Step 3: 関連情報スキャン（70秒）
```
□ 作業に応じた確認
  - プロトコル作業 → protocols/README.md
  - 開発作業 → 該当specifications
  - ルール変更 → CLAUDE.md該当セクション
```

## 🔍 3段階情報階層

### Level 1: 即座把握（status/{agent}.md）
- 現在何をしているか
- 次に何をすべきか
- 何が問題か

### Level 2: 詳細確認（各種README.md）
- 具体的な手順
- 関連情報
- 過去の経緯

### Level 3: 深掘り（個別文書）
- 技術仕様
- 実装詳細
- 過去の議論

## 📊 メモリキャッシュアナロジー

- **L0**: エージェントの作業メモリ（現在処理中）
- **L1**: status/{agent}.md（高速アクセス必要）
- **L2**: records/reports/（中期保存・トレーサビリティ）
- **L3**: その他documents/（長期保存・参照用）

### status/{agent}.mdの理想的サイズ
```
- 現在フェーズ: 10-20行
- 進行中作業: 20-30行
- 作業ログ: 30-50行（直近2-3日分）
- 合計: 200行以内
```

## ✅ 作業中の状況維持

### 30分ごとの確認
```
リマインダー：「現在の作業と次のステップは？」
- 現在のタスク進捗
- 次の具体的アクション
- ブロッカーの有無
```

### 状態変化時の即時更新
- タスク完了 → TodoWrite + status/{agent}.md更新
- 新規発見 → 適切な文書に即座記録
- 問題発生 → ブロッカーとして記録

## 🔄 セッション終了時プロセス（1分）

```
□ status/{agent}.md更新
  - 完了項目追加
  - 次回開始点明記
  - 新規ブロッカー記載

□ 重要な発見の文書化
  - specifications/へ仕様追加
  - protocols/へ新規ルール
  - records/reports/へ作業記録
```

## 📈 成功指標

1. **即答率**: 「現在の作業」を5秒以内に回答95%
2. **認識齟齬**: 10%以下（90%削減）
3. **作業効率**: 40%向上
4. **文脈保持**: セッション間90%以上

## 🚨 更新強制メカニズム

### Git操作時の必須確認
git commit実行前に必ず以下を自問：
1. 「status/{agent}.mdは更新済みか？」
2. 「完了作業は記録したか？」
3. 「次の作業計画は明確か？」

### エスカレーション
- 1回目違反: 警告
- 2連続違反: 作業停止
- 3連続違反: 権限制限
