## 🎯 概要

文書編集・開発作業における基本プロセスの遵守を体系化し、チェックリストと自動リマインダーにより、プロセス違反を95%削減する。メタプロセス強制・インシデント対応強制・チェックリスト強制参照の統合により、包括的なプロセス遵守システムを確立する。

## 📋 3段階プロセス遵守システム

### Level 1: 事前確認（必須）
```
□ status/{agent}.md確認（30秒）
□ 作業に応じた必須文書確認
  - documents編集 → README.md
  - プロトコル関連 → protocols/README.md
  - 開発作業 → protocols/terminology.md
□ LS/Grepによる既存確認
```

### Level 2: 作業中チェック（状況依存）
```
□ 命名規則遵守（file-naming-convention.md参照）
□ 関連文書のリストアップ
□ 更新影響範囲の特定
```

### Level 3: 事後確認（必須）
```
□ README.md更新（構造変更時）
□ 関連文書の参照更新
□ daily記録への記載
```

## 🔔 自動リマインダーシステム

### トリガータイミング
- documents/へのWrite/Edit実行時
- 新規ファイル作成時
- ディレクトリ構造変更時
- 30分経過時の定期確認

### リマインダー内容
```
「基本プロセス確認：
1. 事前確認は完了しましたか？
2. 関連文書は特定しましたか？
3. 更新が必要な文書はありますか？」
```

## 🔧 統合プロセス強制システム

### インシデント対応5フェーズ強制（H027/H038統合）

#### 強制実行フェーズ
1. **Phase 1**: インシデント記録作成（5分以内）
2. **Phase 2**: 5 Whys法による根本原因分析（15分以内）
3. **Phase 3**: 対策立案（20分以内）
4. **Phase 4**: 実装・検証（30分以内）
5. **Phase 5**: フォローアップ・効果測定

#### 強制ワークフロー
```
インシデント検出 → 全作業即座停止 → チェックリスト必須参照 → 5フェーズ完全実行 → 作業復帰
```

### チェックリスト強制参照（H031/H038統合）

#### 対象チェックリスト
- `incident-response-checklist.md`: インシデント対応時
- `hypothesis-creation-checklist.md`: 仮説作成時
- `directory-operation-checklist.md`: ディレクトリ操作時
- `task-completion-checklist.md`: タスク完了時

#### 強制参照メカニズム
```
作業開始 → 「チェックリストを参照しましたか？」→ 未確認時は作業停止
```

### 階層的強制レベル

#### Level 1: 予防的強制（事前防止）
- チェックリスト強制参照
- メタプロセス組み込み
- プロアクティブ警告

#### Level 2: 実行時強制（進行中制御）
- 段階的進行制御
- リアルタイム監視
- 即座修正要求

#### Level 3: 事後強制（完了後検証）
- 完了条件確認
- 品質検証
- エスカレーション連携

## 🚨 違反防止メカニズム

### 強制チェック
- 特定キーワード検出時の一時停止
  - 「新規作成」「ディレクトリ追加」「構造変更」
- 重要ファイル編集時の確認強化
  - README.md、status/{agent}.md、各種ドキュメント

### エスカレーション（H032連携）
- 3回連続スキップで警告レベル上昇
- 重大違反時は作業中断・ユーザー確認
- インシデント対応不備時のH032エスカレーション
- Level 2以上でのインシデント対応手順強制

1. **プロセス違反率**: 5%以下（95%削減）
2. **確認実行率**: 95%以上

4. **構造破損**: 0件

## 💡 運用ルール

### 適用範囲
- すべてのdocuments/編集作業
- 新規ファイル・ディレクトリ作成

### 例外処理
