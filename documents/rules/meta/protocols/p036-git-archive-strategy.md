## 🎯 概要

プロジェクト内にarchiveディレクトリを持たず、Gitのバージョン管理機能を唯一のアーカイブとして使用することで、ディレクトリ構造をシンプルに保ち、検索精度を向上させる。

## 📋 基本方針

### 禁止事項
- ❌ archiveディレクトリの作成
- ❌ old-, backup-, deprecated-等の接頭辞使用
- ❌ 「念のため」の古いファイル保持

### 推奨事項
- ✅ 不要ファイルの即座削除
- ✅ 明確なコミットメッセージ
- ✅ 重要な削除でのタグ使用

## 📊 文書廃止手順

### 標準削除プロセス
```bash
# 1. 廃止理由をコミットメッセージに明記
git rm documents/old-protocol.md
git commit -m "chore: Remove old-protocol.md - superseded by new-protocol.md"

# 2. 重要な廃止の場合はタグ付け
git tag -a v1.0-old-protocol-removed -m "Removed legacy protocol documentation"
```

### コミットメッセージ形式
```
chore: Remove [ファイル名] - [理由]

詳細:
- 代替文書: [新しいファイル名]
- 廃止理由: [具体的な理由]
- 影響範囲: [影響を受ける機能/文書]
```

## 🔍 過去文書の参照方法

### 削除履歴の確認
```bash
# 削除されたファイル一覧
git log --diff-filter=D --summary | grep delete

# 特定ファイルの履歴
git log -- documents/old-protocol.md

# 削除前の内容確認
git show HEAD~10:documents/old-protocol.md
```

### 復元手順
```bash
# 特定コミットからファイル復元
git checkout <commit-hash> -- path/to/file.md

# 最後の削除前の状態に復元
git checkout $(git rev-list -n 1 HEAD -- path/to/file.md)^ -- path/to/file.md
```

## ✅ 運用ルール

### 削除判断基準
1. **即座削除対象**
   - 統合・移行完了した文書
   - 廃止されたプロトコル/仕様
   - 誤って作成されたファイル

2. **慎重削除対象**
   - 参照されている可能性がある文書
   - 履歴的価値のある文書
   - 外部リンクされている文書

### 削除前チェックリスト
```
□ 他の文書からの参照を確認
□ CLAUDE.mdでの言及を確認
□ 代替文書の存在を確認
□ 削除理由を明文化
```
