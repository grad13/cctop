# デプロイメントプロセス仕様

**作成日時**: 2025年6月12日 21:45
**更新理由**: デプロイプロセスに関する認識齟齬の解消

## 🎯 基本原則

**「デプロイ」とは、GitHub経由のCI/CDプロセスを指す。**

### 理由
- CI/CDプロセスの透明性確保
- 評価におけるハック防止
- 自動化されたビルド・デプロイの一貫性

## 📋 デプロイプロセス

### 1. ローカル開発
```bash
npm run dev     # 開発サーバー起動
npm run build   # ビルド確認
```

### 2. GitHubへのプッシュ
```bash
git add -A
git commit -m "feat: 機能説明"
git push origin main
```

### 3. CI/CDによる自動デプロイ
- GitHubへのプッシュがトリガー
- GitHub Actionsが自動的にビルド・デプロイを実行
- 本番環境への反映

### 4. 本番環境での確認
- https://orbital-q.sakura.ne.jp でアクセス
- 動作確認・デバッグ

## ⚠️ 禁止事項

以下の直接的なデプロイ方法は使用しない：
- FTP/SFTPによる直接アップロード
- rsync等による直接転送
- SSHによる直接操作

## 📝 重要な認識

1. **デプロイ = GitHub経由のCI/CD**
2. **手動デプロイは行わない**
3. **全てのデプロイはGitHub Actionsを通じて実行**

## 🔄 ワークフロー

```
開発 → コミット → プッシュ → CI/CD → 本番反映
```

この流れを常に守ることで、透明性と一貫性を保つ。
