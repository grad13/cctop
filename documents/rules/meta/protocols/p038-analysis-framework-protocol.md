## 🎯 概要

分析活動を仮説ベースのフレームワークで体系化することで、知識の蓄積・検証・活用を効率化し、一過性でない継続的な改善サイクルを確立する。

## 📋 分析活動の仮説化プロセス

### 必須プロセス
```
観察・問題発見 → 仮説立案 → 分析実施 → 検証計画 → 運用・改善
```

### プロセス詳細
1. **観察・問題発見**: データ・事象から問題や改善機会を発見
2. **仮説立案**: 検証可能な形で仮説を明文化
3. **分析実施**: データ収集・分析手法適用
4. **検証計画**: 成功指標・測定方法の定義
5. **運用・改善**: 継続的な活用と改善

## 📊 分析種別と対応する仮説タイプ

| 分析種別 | 仮説タイプ | 例 |
|---------|-----------|-----|
| 改善効果分析 | 効果測定仮説 | 「Xの導入により、Y指標がZ%改善する」 |
| パターン分析 | 因果関係仮説 | 「Aが発生する原因はBである」 |
| 成功要因分析 | 再現性仮説 | 「条件C下でDを実施すれば成功する」 |
| 洞察的分析 | 原理仮説 | 「Eの本質はFである」 |

## ✅ 分析品質基準

### 必須要件
1. **検証可能性**: 仮説が測定可能な形で記述されている
2. **再現性**: 同じ方法で分析を再実行可能
3. **実用性**: 具体的な問題解決に活用可能
4. **継続性**: 一過性でなく継続的改善につながる

### 推奨要件
1. **体系性**: 他の分析との関連性が明確
2. **簡潔性**: 本質的な内容に絞られている
3. **更新性**: 定期的な見直しが組み込まれている

## 📈 運用ルール

### 新規分析時
1. 分析開始前に仮説を立案
2. 分析テンプレートに沿って文書化
3. 検証計画を必ず含める
4. records/reports/に配置

### 既存分析の活用
1. 関連する過去分析を参照
2. 検証結果を踏まえて更新
3. 派生分析の必要性を検討

### 定期レビュー
- 月次: 活用頻度と効果測定
- 四半期: 分析体系の見直し
- 年次: フレームワーク自体の改善

## 🚨 アンチパターン

### 避けるべき分析
- ❌ 仮説なき分析（単なるデータ羅列）
- ❌ 検証計画なき分析（やりっぱなし）
- ❌ 独立した分析（関連性不明）
- ❌ 形式的な分析（実用性なし）

### 対策
- ✅ 常に「なぜ」「どう活用するか」を明確に
- ✅ 小さくても検証可能な仮説から開始
- ✅ 既存知識との関連を意識
- ✅ 実際の問題解決を目的とする

## 🔄 改善サイクル

1. **仮説化率測定**: 新規分析の仮説化率100%維持
2. **検証実施率**: 立案仮説の80%以上で検証実施
3. **知識活用率**: 過去分析の月3回以上の参照・活用
4. **体系化進捗**: 分析間の関連付け増加
