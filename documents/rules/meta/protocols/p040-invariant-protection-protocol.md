## 🎯 概要

不変要素（Dominant/H000等）への保護メカニズムを確立し、アクセス前の強制確認プロセスにより、基盤システムへのセキュリティインシデントを完全に防止する。

## 🚨 絶対不変要素リスト

### 完全保護対象（変更・統合・修正は一切禁止）
```
【絶対不変要素】
- dominants/ディレクトリのすべて
- H000（永続仮説）※プロトコル化後はP000
- CLAUDE.md内のDominantセクション

【変更禁止理由】
これらはシステムの基盤であり、変更は全体の崩壊につながる
```

### 改善可能要素（不変ではない）
```
- documents/README.md → 各プロトコルで管理・改善
- status/{agent}.md → P016/P030で管理・改善
- その他のプロトコル・ドキュメント
```

## 🛡️ アクセス前必須確認プロセス

### Level 1: キーワード検出時の自動停止
以下のキーワードと不変要素の組み合わせを検出したら即座に停止：
- 「統合」「変更」「修正」「削除」「更新」
- 「Dominant」「H000」「P000」「DDD」

### Level 2: 3段階確認プロセス
```
1. 対象確認: 操作対象が不変要素リストに含まれないか確認
2. ユーザー承認: 不変要素の場合、必ずユーザーに確認
3. 理由説明: なぜその操作が必要かを明確に説明
4. 代替案提示: 不変要素を変更せずに済む方法を提案
```

### Level 3: ファイル存在確認の強制手順
```bash
# 必須実行順序（省略禁止）
1. ls でディレクトリ確認
2. grep で関連ファイル検索  
3. protocols/README.md で配置確認
4. それでも見つからない場合のみ「存在しない」と判断
```

## ⚠️ 違反時の対応

### 即座停止トリガー
1. **不変要素への操作試行**
   ```
   🚨 停止: [不変要素名]への[操作]は禁止されています。
   この要素はシステムの基盤であり、変更できません。
   ```

2. **確認プロセスのスキップ**
   ```
   ⚠️ 警告: ファイル存在確認プロセスが完了していません。
   必須手順を実行してください。
   ```

### エスカレーション
- 1回目: 警告表示・プロセス再確認
- 2回目: 作業強制停止・ユーザー報告
- 3回目: セキュリティインシデントとして記録
