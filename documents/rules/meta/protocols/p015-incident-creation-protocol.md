## 1. 概要

インシデント記録作成時の手順を標準化し、命名規則の遵守と記録品質を確保する。H041の命名規則標準化を完全に統合し、検索性・管理性を向上させる。

## 2. 命名規則（絶対遵守）

### フォーマット
```
INC-YYYYMMDD-XXX-title.md
```

### 各要素の詳細
- **INC**: 固定接頭辞（大文字必須）
- **YYYYMMDD**: 発生日（例：20250615）
- **XXX**: 3桁の通し番号（最新番号はREADME.md確認）
- **title**: 簡潔な説明（kebab-case、20文字以内推奨）

### titleの作成ルール
1. **必須**: titleは省略不可
2. **形式**: kebab-case（小文字、ハイフン区切り）
3. **長さ**: 5-20文字推奨
4. **内容**: インシデントの本質を表す簡潔な表現

### title例
- ✅ 良い例:
  - `coder-self-resolution`
  - `monitor-h025-violation`
  - `timebox-timer-bug`
- ❌ 悪い例:
  - `bug` （短すぎる）
  - `coder-agent-violated-h025-protocol-three-times` （長すぎる）
  - `CoderViolation` （形式違反）

## 3. インシデント作成手順

### Step 1: 番号確認（必須）
```bash
# 最新番号確認
grep "最新番号:" documents/rules/meta/incidents/README.md
# または
ls documents/rules/meta/incidents/INC-*.md | tail -1
```

### Step 2: ファイル名決定
1. 最新番号 + 1を採番
2. 発生日を確認（YYYYMMDD形式）
3. titleを20文字以内で作成
4. 完全なファイル名を構成

### Step 3: ファイル作成
```bash
# 例：最新が031の場合
cp documents/rules/meta/incidents/incident-template.md \
   documents/rules/meta/incidents/INC-20250615-032-brief-description.md
```

### Step 4: 内容記入
インシデントテンプレートに従って記入

### Step 5: README.md更新
- 最新番号を更新
- インシデント一覧に追加

## 5. よくあるミスと対策

### ミス1: title部分の省略
- **問題**: `INC-20250615-001.md`のようにtitleなし
- **対策**: ファイル作成前にtitleを必ず決定

### ミス2: 番号重複
- **問題**: 既存番号を再利用
- **対策**: 必ずREADME.mdで最新番号確認

### ミス3: 日付形式誤り
- **問題**: `INC-2025-06-15-001`のようにハイフン入り
- **対策**: YYYYMMDDの8桁数字形式を厳守

## 6. エージェント別の注意事項

### Coder Agent
- バグ対応時はbugs/とincidents/の使い分けに注意

### Inspector Agent
- 統計・監視関連のインシデントは`monitor-`接頭辞推奨
- H025等の違反はtitleに仮説番号を含める

### Clerk Agent
- 文書管理・プロセス違反は明確にtitleに反映
- インシデント作成後のREADME.md更新を忘れずに

## 8. 連番管理（H041由来）

### 8.1 連番ルール
- 連番は日付リセットなしの通し番号
- 000から開始、3桁固定
- 最新番号はincidents/README.mdで管理

### 8.2 リスクと対策
- **リスク**: 既存参照の破壊、移行作業のミス、連番管理の煩雑さ
- **対策**: 全参照箇所の網羅的確認（grep使用）、段階的移行、README.mdでの番号管理
