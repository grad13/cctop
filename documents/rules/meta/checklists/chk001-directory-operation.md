# ディレクトリ操作チェックリスト

**作成日時**: 2025年6月14日 09:40  
**目的**: ディレクトリ内容変更時のREADME.md更新を確実にする  

## 📁 README.md = ディレクトリのメタデータ

各ディレクトリのREADME.mdは、macOSの.DS_Storeのように、そのディレクトリの状態を反映する**メタデータファイル**として扱います。

## ✅ ディレクトリ操作時の必須手順

### ファイル追加時
- [ ] ファイルを追加
- [ ] **即座にREADME.md更新**
  - [ ] ファイル一覧に追加
  - [ ] 最終更新日を更新
  - [ ] 必要に応じて説明を追加

### ファイル削除時
- [ ] ファイルを削除
- [ ] **即座にREADME.md更新**
  - [ ] ファイル一覧から削除
  - [ ] 最終更新日を更新
  - [ ] 削除理由を記載（重要な場合）

### ファイル改名時
- [ ] ファイルを改名
- [ ] **即座にREADME.md更新**
  - [ ] 旧名を新名に変更
  - [ ] 最終更新日を更新
  - [ ] 改名理由を記載（必要に応じて）

### ファイル移動時
- [ ] **移動元のREADME.md更新**
  - [ ] ファイル一覧から削除
  - [ ] 移動先を記載
- [ ] **移動先のREADME.md更新**
  - [ ] ファイル一覧に追加
  - [ ] 移動元を記載

## ⚠️ よくあるパターンと対策

### ❌ アンチパターン
1. **バッチ更新**: 複数操作後にまとめて更新
2. **遅延更新**: 「後で更新」と先送り
3. **省略**: 「小さな変更だから」とスキップ

### ✅ ベストプラクティス
1. **即時更新**: 操作と同時に更新
2. **テンプレート活用**: README.mdの形式を統一
3. **自動化意識**: 「操作 = 更新」を習慣化

## 📝 README.md更新テンプレート

```markdown
**最終更新**: YYYY年MM月DD日 HH:MM  
**更新内容**: [何を追加/削除/変更したか]

## 📁 ファイル一覧

| ファイル名 | 説明 | 最終更新 |
|-----------|------|---------|
| example.md | 説明文 | YYYY-MM-DD |
```

## 🔄 このルールの効果

1. **トレーサビリティ向上**: いつ何が変更されたか即座に分かる
2. **管理漏れ防止**: README.mdを見れば現状が把握できる
3. **習慣化促進**: ファイル操作の一部として自然に実行
