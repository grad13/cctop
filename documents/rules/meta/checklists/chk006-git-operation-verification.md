**適用対象**: 全エージェント・全git操作前

## 基本確認項目

### 🎯 **Phase 1: 環境確認**
- [ ] **現在ディレクトリ確認**: `pwd`で位置確認済み
- [ ] **対象ファイル確認**: 変更・追加するファイルのパス明確
- [ ] **git状態確認**: `git status`で現在の状況把握済み

### 📁 **Phase 2: P045判定**
- [ ] **判定ルール適用**: ファイルパスでgit責任範囲を判定済み
- [ ] **親git対象確認**: documents/, passage/, CLAUDE.md, VERSIONs/
- [ ] **子git対象確認**: cctop/src/, cctop/test/, cctop/package.json等
- [ ] **境界違反なし**: 他git管理ファイルが混入していない

### ⚠️ **Phase 3: 境界確認**
- [ ] **正しいディレクトリ**: 適切なgitリポジトリで作業中
- [ ] **ファイル範囲適切**: 管理責任範囲内のファイルのみ対象
- [ ] **.gitignore確認**: 除外ファイルが適切に設定済み

## 頻出ミス・注意点

### 🚨 **よくある間違い**
1. **ディレクトリ混同**: 親gitでcctop/操作・子gitでdocuments/操作
2. **相対パス混乱**: ../で他git領域にアクセス
3. **一括追加**: `git add .`で境界を越えてファイル追加

### 💡 **予防策**
1. **pwd常時確認**: 作業前に必ず現在位置確認
2. **明示的パス**: 相対パスより絶対パス・明示的パス使用
3. **個別追加**: `git add .`より個別ファイル指定
