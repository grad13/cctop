# タスク完了チェックリスト

**作成日時**: 2025年6月14日 05:45  
**最終更新**: 2025年6月14日 06:20  
**目的**: タスク完了時の必須プロセスを確実に実行するため  
**関連**: H017（プロジェクト状況把握プロセス）、H024（タスク完了リマインダー）

## ✅ タスク完了時の必須チェックリスト

### 1. コード関連
- [ ] コード実装が完了している
- [ ] テストを実行した（該当する場合）
- [ ] 不要なデバッグコードを削除した

### 2. ドキュメント更新
- [ ] documents/agents/status/{agent}.mdを更新した
  - [ ] 最終更新日時を更新
  - [ ] 現在の作業フェーズを更新
  - [ ] 完了した作業を記載
- [ ] 関連する仕様書を更新した（仕様変更の場合）
- [ ] 用語集（glossary.md）を更新した（新概念追加の場合）

### 3. 作業記録の整理
- [ ] **完了した作業の詳細をrecords/reports/に切り出した**
  - [ ] 作業内容の詳細
  - [ ] 学んだ教訓
  - [ ] 実装の詳細（コード例など）
- [ ] status/{agent}.mdから詳細を削除し、reportsへの参照に置き換えた

### 4. インシデント・仮説管理
- [ ] 発生したインシデントを記録した（該当する場合）
- [ ] 新しい仮説を作成した（プロセス改善案がある場合）
- [ ] 既存仮説の効果を評価した（該当する場合）

### 5. Git操作（該当する場合）

#### 5.1 コミット前の必須確認
- [ ] **documents/agents/status/{agent}.mdを必ず更新した**（未更新でのコミットはインシデント）
  - [ ] 最終更新日時を更新
  - [ ] 現在の作業フェーズを更新
  - [ ] 完了した作業を「最近完了した作業」に追加
  - [ ] 次の作業計画を更新
  - [ ] ブロッカー・懸念事項を更新（ある場合）
- [ ] 関連するdocumentsを更新した（仕様変更の場合）
- [ ] 完了した作業をrecords/reports/に切り出した（該当する場合）

#### 5.2 コミットメッセージ
- [ ] 適切なコミットメッセージを作成した
- [ ] 関連するissue番号を含めた
- [ ] 破壊的変更の場合は明記した

#### 5.3 推奨コミット手順
```bash
# 1. このチェックリストを確認
# 2. documents/agents/status/{agent}.mdを更新
git add -A
git commit -m "適切なメッセージ"
git push
```

⚠️ **重要**: documents/agents/status/{agent}.md未更新でのコミットはインシデント扱いとなります

## 📝 使用方法

1. **タスク完了時にこのチェックリストを確認**
2. **各項目を実施しながらチェック**
3. **すべてチェックが付いたら完了**

## ⚠️ 重要な注意事項

### reports切り出しのタイミング
- 機能実装が完了した時
- 問題が解決した時
- 重要な調査が終わった時
- その日の作業を終える時

### documents/agents/status/{agent}.mdとrecords/reports/の使い分け
- **documents/agents/status/{agent}.md**: 現在進行中の作業、今後の予定
- **records/reports/**: 完了した作業の詳細記録、学んだ教訓

## 🔄 定期的な見直し

このチェックリストは定期的に見直し、プロジェクトの状況に応じて更新します。
