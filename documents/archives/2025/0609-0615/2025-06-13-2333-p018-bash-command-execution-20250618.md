# Bashコマンド実行前確認プロセス

**Protocol ID**: P018  
**作成日**: 2025年6月13日 nodata  
**更新日**: 2025年6月15日（D001から移行）  
**カテゴリ**: プロセス改善  
**適用頻度**: Bashコマンド実行時（必須）

## 📋 概要

Bashコマンド実行前にCLAUDE.mdの該当セクションを確認することで、ルール違反を完全に防止する確立されたプロセス。

## 🎯 適用条件

以下のいずれかに該当する場合、本プロセスを必ず適用：
- gitコマンドの実行
- 複数コマンドの連続実行を検討
- シェルスクリプトの作成・実行
- システムコマンドの実行全般

## ✅ 具体的手順

### 1. 事前確認（30秒）
```
1. CLAUDE.mdの「Bashコマンド実行ルール」セクションを確認
   - 通常168-184行目付近（構造変更時は検索）
   - 特に「&&使用禁止」ルールを意識

2. 実行予定コマンドのルール準拠性をチェック
   - コマンドチェーン（&&）の有無
   - 個別実行への分解可能性
   - エラーハンドリングの必要性

3. ルール違反があれば修正
   - && → 個別コマンドに分解
   - ; → 個別実行に変更
   - || → 条件分岐で対応
```

### 2. 実行パターン例
```bash
# ❌ 禁止例
git add -A && git commit -m "message" && git push

# ✅ 推奨例
git add -A
git commit -m "message"
git push
```

## ⚠️ 注意事項

### 運用上の留意点
1. **形骸化防止**: 確認作業が機械的にならないよう意識
2. **構造変更対応**: CLAUDE.md更新時は該当箇所を再確認
3. **例外なし**: 緊急時でも必ず実施（30秒の投資で問題防止）

### よくある誤解
- 「簡単なコマンドは確認不要」→ すべてのBashコマンドで必須
- 「慣れたら省略可」→ 習慣化のためにも継続実施
- 「エイリアスなら問題ない」→ エイリアス内容も確認対象
