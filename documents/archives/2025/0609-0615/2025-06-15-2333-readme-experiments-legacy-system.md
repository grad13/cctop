# Experiments - 実験記録

**最終更新**: 2025年6月15日  
**管理**: 全Agent（Coder/Clerk/Monitor）  

## 📋 概要

このディレクトリは、新しい手法・アプローチ・技術の実験記録を管理します。
仮説検証前の予備実験や、探索的な技術検証を記録し、成功した実験は仮説として昇格させます。

## 🔢 実験番号体系（EXP-）

### 命名規則
```
EXP-YYYYMMDD-XXX-title.md
```

- **EXP**: 実験記録の識別子
- **YYYYMMDD**: 実験開始日（例: 20250615）
- **XXX**: 3桁の通し番号（001から開始、全体通し番号）
- **title**: 実験内容の簡潔な説明（kebab-case）

### 命名例
- `EXP-20250615-001-records-performance-test.md`
- `EXP-20250615-002-git-stats-optimization.md`
- `EXP-20250616-003-parallel-processing-trial.md`

## 🎯 実験の目的

1. **新手法の検証**: 仮説として確立する前の予備実験
2. **技術的実現性**: 新技術・ツールの適用可能性検証
3. **パフォーマンス測定**: 既存手法との比較実験
4. **リスク評価**: 変更による影響の事前確認

## 📝 実験記録フォーマット

新規実験ファイルは以下の構成で記録：

```markdown
# [実験タイトル]

**実験番号**: EXP-YYYYMMDD-XXX  
**開始日**: YYYY年MM月DD日  
**実験者**: [Agent名]  
**カテゴリ**: [技術検証/手法改善/パフォーマンス等]  

## 🎯 実験目的
[なぜこの実験を行うのか]

## 📋 実験方法
[どのように実験を実施するか]

## 📊 実験結果
[測定データ・観察結果]

## 🔍 考察
[結果から得られた知見]

## 📌 結論
[成功/失敗の判定と次のアクション]

## 🔄 後続アクション
- [ ] 仮説として立案（H番号: ）
- [ ] 追加実験の実施（EXP番号: ）
- [ ] 本番適用の検討
```

## 🔄 実験のライフサイクル

1. **立案**: 実験の目的・方法を明確化
2. **実施**: 計画に基づいて実験実行
3. **記録**: 結果と考察を詳細に記録
4. **評価**: 成功・失敗の判定
5. **昇格**: 成功した実験は仮説（H番号）として正式化

## 📁 サブディレクトリ構造

実験が複数ファイルにまたがる場合：
```
EXP-YYYYMMDD-XXX-title/
├── README.md        # 実験概要
├── results/         # 実験結果
├── data/           # 実験データ
└── backup/         # バックアップ
```

## ⚠️ 注意事項

1. **番号重複防止**: 新規作成時は必ず最新番号を確認
2. **削除時の扱い**: 失敗実験も記録として保持（削除しない）
3. **仮説との関係**: 昇格した場合は相互参照を明記
4. **共同実験**: 複数Agentが関わる場合は明記
