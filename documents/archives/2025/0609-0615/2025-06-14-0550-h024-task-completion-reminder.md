# H024: タスク完了時チェックリスト自動リマインダー

**仮説ID**: H024  
**作成日時**: 2025年6月14日 05:50  
**カテゴリ**: Process（プロセス改善）  
**ステータス**: 提案  
**優先度**: 中  
**背景**: INC-20250614-005（daily切り出し忘れ）

## 🎯 問題認識

### 現状の問題
1. **タスク完了時にdaily切り出しを忘れる**
   - 問題解決の達成感で次のプロセスを忘れる
   - status更新で満足してしまう

2. **チェックリストの存在を忘れる**
   - 作成したが使われない可能性
   - 習慣化されるまでの課題

## 💡 仮説内容

**「タスク完了を示すキーワードで自動的にチェックリストを提示することで、プロセス実行を確実にできる」**

## 🔧 実装案

### トリガーとなるキーワード
- 「解決」「完了」「終了」「できました」
- 「fix:」「feat:」（コミットメッセージ）
- 「✅」「完全解決」

### 自動リマインダーの内容
```
📋 タスク完了チェックリスト確認
documents/rules/meta/checklists/task-completion-checklist.md

特に重要：
- [ ] status/{agent}.md更新
- [ ] dailyへの切り出し
- [ ] 関連ドキュメント更新

すべて完了しましたか？
```

### 実装方法
1. コンソール出力での定期リマインド
2. キーワード検出時の自動表示
3. git commit前の確認（既存の仕組み活用）

## 📊 期待効果

- daily切り出し忘れ: 0件
- チェックリスト使用率: 100%
- プロセス完全実行率の向上
