# H022: アクティブリスニングプロトコル

**仮説ID**: H022  
**作成日時**: 2025年6月14日 04:25  
**カテゴリ**: Communication（コミュニケーション改善）  
**ステータス**: 実施中  
**優先度**: 最高  
**背景**: INC-20250614-002（コミュニケーション障害）の教訓

## 🎯 問題認識

### 現状の重大な問題
1. **ユーザー説明を正しく理解しない**
   - 自分の技術的解釈を優先
   - ユーザーの言葉を軽視

2. **確認をせずに進める**
   - 思い込みで作業開始
   - 誤解に気づくのが遅い

3. **「再三」の指摘を受ける**
   - 同じ説明を何度もさせる
   - ユーザーのフラストレーション増大

## 💡 仮説内容

**「ユーザー説明の要約・確認を体系的に行うことで、コミュニケーション齟齬を完全に防げる」**

## 🔧 実装内容

### 1. 理解確認テンプレート

```markdown
## 理解確認

私の理解を確認させてください：

**問題の本質**：
- [ユーザーの言葉をそのまま引用]
- [自分の言葉での要約]

**重要なポイント**：
1. [ポイント1]
2. [ポイント2]
3. [ポイント3]

**確認事項**：
- [不明な点や曖昧な表現]

これで正しいでしょうか？
```

### 2. トリガーワード検出

以下の言葉が出たら**即座に理解の見直し**：
- 「再三」
- 「何度も」
- 「さっき言った」
- 「違う」
- 「そうじゃない」
- 「聞いてる？」
- 「真面目に」

### 3. 問題定義書の作成

技術作業開始前に必ず作成：

```markdown
## Quick Switch問題定義書

**ユーザーの言葉**：
「最初にガチャガチャ移動すると、継続してquick switchできる」

**問題の解釈**：
- 初回操作：失敗する
- 複数回操作後：正常動作
- つまり：最初の1回だけの問題

**確認済み事項**：
- どの島から始まっても同じ ✓
- タイマー依存ではない ✓
- 一度動けば継続的に動作 ✓

**調査方針**：
初回キーイベントの処理に焦点
```

### 4. コミュニケーションログ

重要な説明は必ず記録：

```markdown
## 2025-06-14 コミュニケーションログ

**時刻**: 04:15
**ユーザー説明**: 「ガチャガチャ移動すると動く」
**私の理解**: 複数回操作で動作開始
**確認**: ✓ ユーザー承認
```
