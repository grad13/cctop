# 仮説: メタプロセス改善フレームワーク

**仮説ID**: H003
**作成日時**: 2025年06月12日 20:25
**ステータス**: 継続検証（改良版）
**検証期間**: 2025年06月12日 ～ 2025年06月20日（延長）

## 🎯 仮説の内容

### 仮説文
**「重要作業開始前にCLAUDE.md全体を確認することで、既存プロセス・ルールの見落としを防げる」**

### 背景
- 仮説立案時にCLAUDE.mdの既存仮説プロセス記載を見落とし
- 自分で気付くべき情報を「後追い発見」する非効率性
- メタレベルでの作業品質管理が不十分

### 検証対象問題  
仮説検証プロセス文書見落とし事案（2025年06月12日発生）

## 🔬 検証方法

### 改善プロセス
```
【問題手順】
仮説立案 → 検証方法検討 → 文書作成 → 指摘受ける → 既存文書発見

【改善手順】
重要作業前 → CLAUDE.md全体確認 → 既存情報チェック → 仮説立案 → 検証実行
```

### 改良版チェックリスト（3段階アプローチ）

#### 第1段階: 高速スキャン（2分以内）
1. **CLAUDE.md目次チェック**
   - 関連セクションの特定（Ctrl+Fで検索）
   - 仮説管理セクションの確認
   - 現在実施中の仮説リスト確認

2. **作業分類判定**
   - 仮説関連 → hypotheses/README.md確認
   - ドキュメント関連 → GUIDELINES.md確認
   - 開発ルール関連 → rules/確認

#### 第2段階: 詳細確認（3分以内）
1. **特定ディレクトリの精査**
   - 作業分類に応じた重点確認
   - 既存文書のタイトルスキャン
   - 類似作業の前例確認

2. **キーワード検索**
   - Grepによる関連文書の特定
   - 重複可能性のある内容確認

#### 第3段階: 作業方針決定（1分以内）
1. **判定フローチャート**
   - 既存文書で解決可能 → 参照して終了
   - 既存文書の拡張で対応 → 編集作業へ
   - 新規作成が必要 → 配置場所をGUIDELINES.md確認

2. **最終チェック**
   - 作業の重複がないか
   - より上位の解決策がないか

### 対象作業
- 仮説立案・検証
- 新機能設計
- 問題解決プロセス
- ドキュメント作成

### 測定指標
- **見落とし発生件数**: 0件が目標
- **事前確認実行率**: 100%が目標  
- **確認時間**: 10分以内/作業
- **効率向上**: 重複作業の削減

## 📊 評価基準

### 成功条件
- 見落とし: 0件 + 確認実行率: 100% + 確認時間: 10分以内 + 重複作業: 大幅削減

### 部分成功条件
- 見落とし: 大幅減少 + 確認実行率: 向上 + 確認時間: 許容範囲

### 失敗条件  
- 見落とし: 継続発生 OR 確認時間: 過大 OR 確認実行率: 低下

## 🔄 仮説更新ルール

### 失敗時の見直しパターン
1. **確認時間が過大**: より効率的なスキャン方法を検討
2. **確認が形骸化**: チェックリスト・自動化を検討
3. **既存情報の発見困難**: 文書構造・命名規則の見直し

### 成功時の展開
- 標準作業プロセスへの組み込み
- 他作業への適用拡大
- 自動リマインダー機能の検討
