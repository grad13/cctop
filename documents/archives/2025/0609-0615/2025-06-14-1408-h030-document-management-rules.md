## 📏 ドキュメント管理・組織化ルール

### 必須：文書配置ガイドライン

**すべての文書作成・移動時に以下のルールを厳格に適用すること**：

#### 📁 ディレクトリ役割分担
```
documents/
├── specifications/  # 現在のシステム定義（What it is）
│   ├── architecture/  - 実装済みアーキテクチャ
│   ├── business/      - 確定したビジネス戦略
│   └── [機能名]/      - 実装完了機能の仕様
├── roadmaps/       # 将来の計画・議論（How to build）
│   ├── [機能名]/     - 機能別開発計画・議論
│   └── project-milestones.md
├── meta/protocols/ # 開発ルール（How to work）
│   ├── terminology.md     - 命名規則・実装ルール
│   ├── glossary.md        - プロジェクト用語集
│   └── development.md
├── daily/          # 作業記録（What was done）
│   └── YYYY-MM-DD-HHMM-*.md
└── archive/        # 過去の参考資料（Historical reference）
    ├── YYYY-MM/    - 月別アーカイブ
    └── milestones/ - 完了マイルストーン
```

#### 🎯 配置決定フローチャート
1. **現在の状態を説明する** → `specifications/`
2. **将来の実装計画・議論** → `roadmaps/[機能名]/`
3. **開発・作業のルール** → `meta/protocols/`
4. **その日の作業記録** → `daily/`
5. **過去の参考資料** → `archive/`

#### ❌ 禁止事項
- **重複概念のディレクトリ作成禁止**
  - `questions/` と `roadmaps/` の重複
  - `completed/` と `archive/` の重複
  - `policies/` と `rules/` の重複
- **曖昧な配置の禁止**
  - 「どちらにも該当しそう」な場合は必ずユーザーに確認
- **命名規則違反の禁止**
  - 必ず `YYYY-MM-DD-HHMM-description.md` 形式を使用

### 必須：文書作成時チェックリスト

**新規文書作成時は必ず以下を確認**：

1. **配置場所の確認**
   - [ ] ディレクトリ役割分担に従って適切な場所に配置
   - [ ] 類似内容の既存文書がないか確認
   - [ ] 関連文書とのリンク整合性確認

2. **命名規則の確認**
   - [ ] `YYYY-MM-DD-HHMM-description.md` 形式
   - [ ] description部分は簡潔で内容を表現
   - [ ] 既存ファイルとの重複名回避

3. **内容整合性の確認**
   - [ ] 用語使用が `meta/protocols/terminology.md` と一致
   - [ ] 関連する `specifications/` 文書と矛盾なし
   - [ ] 必要に応じて `documents/README.md` にリンク追加

### 必須：プロジェクト整理時の網羅チェック

**プロジェクト整理・クリーンアップ時は以下を必ず実行**：

1. **構造整合性チェック**
   - 全ディレクトリ名が役割分担と一致しているか
   - 重複概念のディレクトリが存在しないか
   - ファイル配置が適切なディレクトリにあるか

2. **命名規則チェック**
   - 全ファイルが `YYYY-MM-DD-HHMM-description.md` 形式か
   - ディレクトリ名が統一されているか
   - 参照リンクが正しいパスを指しているか

3. **内容重複チェック**
   - 同じ内容が複数箇所にないか
   - 古い情報と新しい情報が混在していないか
   - 参照リンクが全て有効か

### 定期メンテナンス義務

1. **週次チェック**（新機能開発開始時）
   - 重複ファイル・ディレクトリの確認
   - リンク切れの検出・修正
   - 命名規則違反の修正

2. **月次整理**（マイルストーン完了時）
   - 完了した `roadmaps/` 項目を `specifications/` に移動
   - 古い `daily/` ログを `archive/` に移動
   - 不要になったファイルの削除

## 📝 documents/README.mdメンテナンス責任

**documents/README.mdは documents管理の司令塔として、常に最新かつ正確である必要があります。**

### 📋 メンテナンス必須タイミング

3. **編集パターン追加時**: 必要資料マップへの新パターン追加
4. **プロセス改善時**: チェックリストの見直し・更新

- **網羅性**: 全editing scenarioをカバー
- **正確性**: リンク切れ・パス誤りゼロ
- **最新性**: 構造変更の即日反映
- **効率性**: 必要最小限の確認事項に絞り込み
