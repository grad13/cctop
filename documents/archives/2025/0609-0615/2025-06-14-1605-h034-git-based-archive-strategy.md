# H034: Git完結型アーカイブ戦略

**作成日時**: 2025年6月14日 16:05  
**検証期間**: 2025年6月14日〜6月21日（1週間）  
**ステータス**: 検証中  
**カテゴリ**: document-management  

## 🎯 仮説文

プロジェクト内にarchiveディレクトリを持たず、Gitのバージョン管理機能を唯一のアーカイブとして使用することで、ディレクトリ構造がシンプルになり、混乱が減少し、メンテナンスが容易になる。

## 📋 背景

### 問題の詳細
- **現象**: archiveディレクトリに古い文書が蓄積
- **影響**: 検索時に古い文書がヒット、誤参照のリスク
- **規模**: 112KBのarchiveディレクトリが存在

### 問題分析
1. **重複管理**: Gitとarchiveディレクトリの二重管理
2. **混乱の元**: 古い仕様やルールが検索結果に出現
3. **メンテナンス負担**: archiveへの移動判断が必要

## 🔬 検証方法

### 実装内容
1. **archiveディレクトリの完全削除**
   - 既存のarchiveディレクトリを削除
   - 今後もarchiveディレクトリを作成しない

2. **Git活用方針**
   - 不要なファイルは即座に削除
   - 過去の状態はgit logで確認
   - 必要時はgit checkoutで復元

3. **運用ルール**
   - 廃止文書は削除（archiveではなく）
   - 重要な変更はコミットメッセージに明記
   - タグ機能で重要なマイルストーンを記録

### 測定指標
- **検索精度**: 古い文書の誤参照件数（目標: 0件）
- **ディレクトリ整理度**: 不要ディレクトリ数（目標: 0）
- **復元成功率**: 必要時のgit復元成功率（目標: 100%）

## 📝 実装例

### 文書廃止時の手順
```bash
# 1. 廃止理由をコミットメッセージに明記
git rm documents/old-protocol.md
git commit -m "chore: Remove old-protocol.md - superseded by new-protocol.md"

# 2. 重要な廃止の場合はタグ付け
git tag -a v1.0-old-protocol-removed -m "Removed legacy protocol documentation"
```

### 過去文書の参照方法
```bash
# 1. 削除履歴の確認
git log --diff-filter=D --summary | grep delete

# 2. 特定ファイルの履歴確認
git log -- documents/old-protocol.md

# 3. 過去の状態を確認
git show HEAD~10:documents/old-protocol.md
```
