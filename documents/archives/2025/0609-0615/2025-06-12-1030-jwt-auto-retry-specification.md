# JWT自動リトライ機能仕様書

**作成日時**: 2025年06月12日 10:30
**目的**: TimeBox-TaskGrid連携における401エラー自動回復機能の設計
**ステータス**: 仕様策定中

## 問題の背景

### 現在の状況
- ログアウト後の再guest access時も401エラーが継続発生
- 手動でのJWT更新では正常動作することを確認
- 既存の自動リトライ機能が不完全

### 技術的原因
1. **JWT署名検証問題**: 古いJWTトークンの署名が無効
2. **自動リトライの不備**: 1回のリトライ後の処理に問題
3. **UIフィードバック不足**: ユーザーに状況が伝わらない

## 要件定義

### 機能要件

#### FR1: JWT自動検証・更新
- **説明**: 401エラー検出時にJWTの自動検証と更新を行う
- **動作**: API呼び出し前にJWT有効性をチェック、必要に応じて更新
- **対象**: ゲストユーザーのみ（本登録ユーザーは手動ログイン必要）

#### FR2: 段階的リトライ
- **説明**: 複数段階でのリトライ機能
- **段階1**: JWT更新後の即座リトライ
- **段階2**: 短時間待機後の再試行
- **段階3**: フォールバック（サンプルデータ表示）

#### FR3: ユーザーフィードバック
- **説明**: リトライ中の適切なUI表示
- **Loading状態**: 「Connecting to TaskGrid...」
- **Progress表示**: リトライ回数と進捗
- **Error通知**: 最終的な失敗時の案内

#### FR4: ログ記録
- **説明**: トラブルシューティング用のログ記録
- **記録内容**: JWT状態、API応答、リトライ回数
- **出力先**: コンソールログ（本番では最小限）

### 非機能要件

#### NFR1: パフォーマンス
- **リトライ間隔**: 500ms, 1s, 2s の段階的増加
- **最大リトライ回数**: 3回まで
- **タイムアウト**: 各API呼び出しは5秒以内

#### NFR2: ユーザビリティ
- **透明性**: リトライはユーザーに負担をかけない
- **回復性**: 一時的な問題からの自動回復
- **明確性**: 恒久的な問題時の明確なメッセージ

#### NFR3: 保守性
- **設定可能**: リトライ回数・間隔の調整可能
- **テスト容易性**: 各段階の個別テスト可能
- **ログ制御**: 本番環境での適切なログレベル

## 技術設計

### アーキテクチャ

```
TimeBox loadTodos()
    ↓
JWT Validation
    ↓ (invalid)
JWT Refresh
    ↓
API Retry Manager
    ↓
├─ Retry 1: Immediate
├─ Retry 2: 500ms wait
├─ Retry 3: 1s wait
└─ Fallback: Sample data
```

### クラス設計

```javascript
class JWTRetryManager {
  constructor(maxRetries = 3, baseDelay = 500) {}
  
  async executeWithRetry(apiCall, context) {}
  async refreshJWT() {}
  calculateDelay(attempt) {}
  logRetryAttempt(attempt, error) {}
}
```

### API設計

```javascript
// 使用例
const retryManager = new JWTRetryManager();
const todos = await retryManager.executeWithRetry(
  () => apiClient.loadTaskGridData(),
  { component: 'TimeBox', operation: 'loadTodos' }
);
```

## テストケース

### TC1: 正常なJWT更新
- **前提**: 期限切れJWTがlocalStorageに存在
- **操作**: TimeBox初期化
- **期待**: 新しいJWT取得後、TaskGridデータ表示

### TC2: JWT更新失敗
- **前提**: ネットワーク問題でJWT更新不可
- **操作**: TimeBox初期化
- **期待**: リトライ後、サンプルデータ表示

### TC3: ログアウト後の再アクセス
- **前提**: ログアウト実行済み
- **操作**: guest access→TimeBox表示
- **期待**: 新しいJWTで正常なデータ表示

### TC4: 連続401エラー
