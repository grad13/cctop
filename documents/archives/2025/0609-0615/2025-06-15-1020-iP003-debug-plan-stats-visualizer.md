# 統計ビジュアライザー デバッグ計画書

作成日時: 2025年6月15日 10:20  
作成者: Monitor Agent

## 1. 問題の概要

統計ビジュアライザーで以下の症状が発生：
- リロード時にグラフが表示されない
- 「全て選択」状態でもデータが表示されない
- 複数回の修正試行後も改善されない

## 2. 調査手順

### Phase 1: 現状把握（10分）
1. **ブラウザコンソールエラーの確認**
   - [ ] stats-visualizer.htmlを開く
   - [ ] 開発者ツールのコンソールを確認
   - [ ] エラーメッセージを記録
   - [ ] ネットワークタブでAPI通信を確認

2. **データフローの確認**
   - [ ] /api/mapping のレスポンス確認
   - [ ] /api/merged-records のレスポンス確認
   - [ ] /api/records のレスポンス確認

### Phase 2: ログによる詳細調査（15分）
1. **既に追加したデバッグログの確認**
   - loadData関数の実行フロー
   - fileMapping の設定状態
   - binaryData の内容
   - updateChart関数の実行状態

2. **追加ログが必要な箇所の特定**
   - [ ] Chart.js初期化部分
   - [ ] データセット作成部分
   - [ ] メトリクス選択状態

### Phase 3: 問題の特定（10分）
1. **エラーパターンの分析**
   - JavaScript実行エラーか？ → コンソールエラーなし
   - データ形式の不一致か？ → **確認済み：fileMappingの逆引きが誤っている**
   - タイミング問題か？ → APIは正常応答

2. **根本原因の仮説立案**
   - **仮説1（確定）**: fileMappingが{filePath: id}形式なのに、fileMapping[fileId]で逆引きしている
   - 仮説2: Chart.js初期化タイミングの問題
   - 仮説3: メトリクス選択状態の初期化問題

### Phase 4: 修正実装（20分）
1. **最小限の修正で検証**
   - 仮説1の検証と修正
   - 動作確認
   - 必要に応じて次の仮説へ

2. **修正の妥当性確認**
   - 他の機能への影響確認
   - エッジケースの考慮

## 3. 期待される結果

- リロード時に正しくグラフが表示される
- 選択したメトリクスのデータが正確に表示される
- エラーなく安定して動作する

## 4. 成功基準

1. ブラウザコンソールにエラーが出ない
2. リロード後も設定が保持される
3. すべてのメトリクスが正しく表示される
