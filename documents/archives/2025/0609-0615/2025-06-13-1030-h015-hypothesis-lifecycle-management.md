# H015: 仮説ライフサイクル管理

**仮説ID**: H015  
**作成日時**: 2025年6月13日 10:30  
**カテゴリ**: Meta（仮説管理プロセス）  
**ステータス**: 提案中  
**優先度**: 高（仮説管理の基盤）

## 🎯 背景

### 発見された問題
1. H001、H003の検証期間が終了したが、評価が行われていない
2. 期限切れ仮説の扱いが不明確
3. CLAUDE.mdの「現在実施中の仮説」が更新されていない

## 🧪 仮説

**「仮説のライフサイクル（作成→検証→評価→昇格/廃棄）を明確化し、期限管理プロセスを導入することで、仮説の有効活用と管理効率を向上できる」**

## 🎯 提案する仕組み

### 1. 仮説のステータス定義
```
1. 【提案中】: 仮説立案済み、検証開始前
2. 【検証中】: 検証期間内でアクティブ
3. 【期限切れ・評価待ち】: 検証期間終了、評価未実施
4. 【成功・昇格待ち】: 成功基準達成、directions昇格待ち
5. 【失敗・アーカイブ待ち】: 成功基準未達、アーカイブ待ち
6. 【永続検証】: H000のような無期限検証
```

### 2. 期限管理プロセス
1. **毎日朝のチェック**: 期限切れ仮説の確認
2. **期限切れ通知**: CLAUDE.mdに【期限切れ・評価待ち】マーク
3. **48時間ルール**: 期限切れから48時間以内に評価実施
4. **自動アーカイブ**: 7日間評価なしの場合、自動的にarchiveへ

### 3. CLAUDE.md更新ルール
- **検証開始時**: 「現在実施中の仮説」に追加
- **期限切れ時**: 【期限切れ・評価待ち】マーク追加
- **評価完了時**: 結果に応じて削除またはdirections参照へ

### 4. 評価テンプレート
```markdown
## 検証結果評価（YYYY年MM月DD日）

### 成功指標の達成状況
- [ ] 指標1: 達成/未達成
- [ ] 指標2: 達成/未達成

### 定量的結果
- 実績値: XX%
- 目標値: XX%

### 定性的評価
- 効果: 
- 課題: 

### 結論
- [ ] 成功 → directionsへ昇格
- [ ] 部分成功 → 継続検証
- [ ] 失敗 → archiveへ
```
