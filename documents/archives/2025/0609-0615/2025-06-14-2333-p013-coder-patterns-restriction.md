# P013: Coderエージェントpatternsコマンド使用禁止プロトコル

**Protocol ID**: P013  
**作成日**: 2025年6月14日 nodata  
**背景**: patternsコマンド使用による思い込み行動・README.md無視問題の対策  
**適用対象**: Coder Agent限定  

## 📋 概要

Coderエージェントによるpatternsコマンド使用を禁止し、適切なREADME.md確認を義務化することで、思い込みに基づく不正確な実装を防ぐ。

## 🚫 禁止事項

### patternsコマンド使用の完全禁止
- **禁止コマンド**: `patterns`, `find . -name "*.pattern"`, パターン検索系コマンド
- **適用範囲**: Coderエージェントのすべての作業時
- **例外**: 一切認めない

### 代替手段の強制
Coderは以下の手順を必ず実行：

1. **README.md優先確認**
   ```bash
   # 必須手順
   find . -name "README.md" | head -5
   cat ./README.md  # 作業ディレクトリのREADME
   cat ./<target-directory>/README.md  # 対象ディレクトリのREADME
   ```

2. **ディレクトリ構造の理解**
   ```bash
   ls -la <target-directory>/
   tree <target-directory>/ -L 2  # 2レベルまでの構造把握
   ```

3. **必要に応じて具体的ファイル確認**
   ```bash
   ls <target-directory>/*.js
   head -20 <target-directory>/main.js  # 実際のファイル内容確認
   ```

## ⚠️ 違反時の対応

### 検出方法
- Coderが以下のコマンドを使用した場合は即座違反
  - `patterns`を含むコマンド
  - パターンファイル検索コマンド
  - README.md確認なしでの実装開始

### エスカレーション
1. **Level 1**: ⚠️ 即座停止・P013再確認・README.md確認義務
2. **Level 2**: 🛑 作業強制停止・Clerkによる手順確認
3. **Level 3**: 🚫 コード実装権限一時制限

## 📈 期待効果

### 直接効果
- **思い込み防止**: パターンに頼らない正確な情報収集
- **README重視**: 適切なドキュメント確認習慣の確立
- **品質向上**: 実際の構造に基づく正確な実装

### 間接効果
- **認識齟齬削減**: ユーザーとの実装方針一致

- **メンテナンス性**: 適切な構造理解に基づく保守可能なコード

## 🔍 実装例

### ❌ 禁止される行動パターン
```bash
# これらは全て禁止
patterns src/
find . -name "*.pattern"
grep -r "pattern" .
# README.mdを読まずにいきなり実装開始
```

### ✅ 推奨される行動パターン
```bash
# 必須手順
cat README.md
ls -la src/
cat src/README.md
ls src/components/
head -20 src/components/main.js

# 理解確認後に実装開始
```
