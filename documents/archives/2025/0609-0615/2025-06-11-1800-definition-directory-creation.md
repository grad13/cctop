# definitionディレクトリの作成と文書整理

**作成日時**: 2025年6月11日 18:00

## 実施内容

### definitionディレクトリの作成

#### 目的
- **現在の定義・仕様**を一元管理
- コードの意図が理解できる参照ドキュメント
- 「What it is」を明確化

#### 実施した移動
```
architecture/ → definition/architecture/
business/    → definition/business/
```

### 役割分担の明確化

#### definition/ - 現在の定義
- 実装済みのシステム仕様
- 現在のアーキテクチャ
- ビジネス戦略

#### roadmap/ - 将来の計画
- 未実装機能の仕様
- 技術的議論・課題
- 改善提案

#### rules/ - 開発ルール
- 命名規則
- コーディング規約
- 作業プロセス

### 利点

1. **明確な分離**
   - 「現在の状態」と「将来の計画」が明確に分離
   - 新規開発者がシステムを理解しやすい

2. **メンテナンス性向上**
   - 実装と仕様の一致を確認しやすい
   - 古い仕様が残りにくい

3. **効率的な参照**
   - コードを理解したい → definition/
   - 機能を追加したい → roadmap/
   - ルールを確認したい → rules/

## 今後の方針

- 実装完了時は、roadmap/ から definition/ に移動
- definition/ は常に最新の状態を保つ
- 議論・検討は roadmap/ で行う
