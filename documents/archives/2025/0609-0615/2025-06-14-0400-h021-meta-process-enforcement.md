# H021: メタプロセス強制実行メカニズム

**仮説ID**: H021  
**作成日時**: 2025年6月14日 04:00  
**カテゴリ**: Meta（メタプロセス改善）  
**ステータス**: 実施中  
**優先度**: 最高  
**背景**: INC-20250614-001（12時間デバッグ失敗）の教訓

## 🎯 問題認識

### 現状の重大な問題
1. **仮説文書を読まない**
   - H013（根本原因分析）を無視して対症療法を繰り返す
   - H020（包括的ログ）を知らずに段階的調査を行う

2. **LEGACY_STATUS.mdを更新しない**
   - 作業状況が不透明になる
   - 次回セッション時に状況把握が困難

3. **プロセスの存在を忘れる**
   - CLAUDE.mdに記載があっても実行されない
   - インシデントが繰り返される

## 💡 仮説内容

**「メタプロセスの実行を技術的に強制することで、人的エラーによるプロセス違反を防げる」**

## 🔧 実装内容

### 1. 自動チェックポイント（即座実装）

```javascript
// quick-switch-debug.js に追加
function enforceMetaProcesses() {
  // 30分タイマー
  setInterval(() => {
    console.warn('⏰ 30分経過 - LEGACY_STATUS.md更新が必要です');
    console.warn('📋 現在の作業内容を記録してください');
  }, 30 * 60 * 1000);
  
  // 1時間タイマー
  setInterval(() => {
    console.error('🚨 1時間経過 - 対症療法チェック');
    console.error('❓ これは根本解決ですか？');
    console.error('📖 H013を読み直してください');
  }, 60 * 60 * 1000);
}
```

### 2. デバッグ開始時の強制確認

```javascript
// すべてのデバッグツールの先頭に追加
```

## 📊 検証指標

### 定量的指標
- プロセス違反数: 0件/週を目標
- LEGACY_STATUS.md更新頻度: 30分に1回以上
- デバッグ成功率: 2時間以内80%

### 定性的指標
- 仮説文書の参照が習慣化される
- 対症療法的アプローチが激減
- インシデントの再発がなくなる

## 🔄 実施計画

### Phase 1: 技術的強制（即座）
- 上記コードをすべてのデバッグツールに実装
- コンソール警告の常時表示

### Phase 2: 習慣形成（1週間）
- 毎回の強制確認により習慣化
- 違反時の即座フィードバック

### Phase 3: 評価と調整（2週間後）
- 効果測定
- 必要に応じて強制レベル調整

## ⚠️ 注意事項

### バランスの重要性
- 過度な強制は作業効率を下げる
- 最小限の介入で最大の効果を狙う

### 段階的緩和

- 最終的には自発的実行を目指す
