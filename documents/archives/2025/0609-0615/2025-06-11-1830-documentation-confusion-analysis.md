# ドキュメント混乱の原因分析と改善策

**作成日時**: 2025年6月11日 18:30

## 🔍 混乱の根本原因

### 1. **明確な配置ガイドラインの欠如**
**問題点**:
- 新しい文書をどこに配置すべきか不明確
- 類似目的のディレクトリが複数存在（completed/archive、questions/roadmap）
- 「現在の状態」と「将来の計画」の混在

**影響**:
- 同じ種類の文書が異なる場所に散在
- 情報の重複と矛盾
- 必要な情報を見つけるのが困難

### 2. **一貫性のない命名規則**
**問題点**:
- 日付フォーマットの不統一（`0610-xxx.md` vs `2025-06-10-xxx.md`）
- ディレクトリ名の不一致（`policies/` vs `rules/`）
- タイムスタンプの有無が混在

**影響**:
- ファイルの時系列追跡が困難
- 自動化ツールでの処理が複雑化
- プロジェクトの専門性が低下

### 3. **チェック体制の不備**
**問題点**:
- プロジェクト全体の整合性チェックが不十分
- 参照リンクの妥当性検証なし
- 重複・矛盾の定期的な確認なし

**影響**:
- 問題が蓄積してから発覚
- 修正作業が大規模化
- ユーザーからの信頼性低下

### 4. **役割分担の曖昧さ**
**問題点**:
- definition（定義）とroadmap（計画）の区別なし
- architecture（設計）とbusiness（戦略）の配置が不明確
- daily（作業ログ）とquestions（課題）の境界が曖昧

**影響**:
- 文書の重複作成
- 更新漏れの発生
- メンテナンス負荷の増大

## 💡 実施した改善策

### 1. **明確なディレクトリ構造の確立**
```
definition/  → 現在のシステム定義（What it is）
roadmap/     → 将来の計画・議論（How to build）
rules/       → 開発ルール（How to work）
daily/       → 作業記録（What was done）
archive/     → 過去の参考資料（Historical reference）
```

### 2. **統一された命名規則**
- 全ファイル: `YYYY-MM-DD-HHMM-description.md`
- ディレクトリ名: 単数形、小文字、ハイフン区切り

### 3. **重複概念の統合**
- questions/ → roadmap/（議論は機能開発の一部）
- completed/ → archive/milestones/（完了は過去の記録）

### 4. **チェックプロセスの導入**
- 定期的な整合性確認
- 自動化可能な部分の特定
- CLAUDE.mdへのルール明記

## 📋 予防策（CLAUDE.mdに追加）

### 文書作成時の必須確認
1. **配置場所の確認**
   - 現在の定義 → definition/
   - 将来の計画 → roadmap/
   - 開発ルール → rules/
   - 作業記録 → daily/

2. **命名規則の遵守**
   - 必ず`YYYY-MM-DD-HHMM-description.md`形式
   - 説明部分は簡潔で明確に

3. **既存文書との整合性**
   - 同様の内容がないか確認
   - 関連文書を更新
   - 参照リンクの妥当性確認

### 定期的なメンテナンス
1. **週次チェック**
   - 重複ファイルの確認
   - リンク切れの検出
   - 命名規則違反の修正

2. **月次整理**
   - 古いdailyログのarchive移動
   - roadmap完了項目のdefinition移動
   - 不要ファイルの削除

## 🎯 期待される効果

1. **開発効率の向上**
   - 必要な情報に素早くアクセス
   - 重複作業の削減
   - 明確な作業フロー

2. **品質の向上**
   - 一貫性のある文書体系
   - 最新情報の維持
   - 信頼性の高い参照資料

3. **協働の円滑化**
   - 新規参加者の理解促進
   - 作業の引き継ぎ簡易化
   - コミュニケーションコスト削減
