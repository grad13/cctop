# H026: マルチエージェントステータス管理システム

## 仮説
複数のエージェント（Coder、Clerk、Monitor）が単一のLEGACY_STATUS.mdを共有することで発生する競合や事故を防ぐため、エージェントごとに独立したステータスファイルを管理する体制が効果的である。

## 背景
- 複数エージェントが同時に作業する際の編集競合リスク
- 各エージェントの作業内容が混在し、可読性が低下
- 権限管理の観点から、エージェントごとの明確な境界が必要

## 提案する解決策

### 1. ディレクトリ構造
```
documents/agents/status/
├── README.md      # ステータス管理の説明
├── coder.md      # Coderエージェントの作業ログ
├── clerk.md      # Clerkエージェントの作業ログ
└── monitor.md    # Monitorエージェントの作業ログ
```

### 2. Daily切り出しプロセス
Clerkエージェントが以下の手順で安全に切り出しを実行：
1. 対象ファイル（f.md）を読み込む
2. dailyディレクトリに内容を切り出す
3. 切り出し後の残りをf_.mdとして一時保存
4. 作業中にf.mdが更新されていた場合、追記情報をf_.mdに統合
5. f_.mdをf.mdに置き換える（f_.mdは削除）

### 3. 権限と責任
- **Coder**: `status/coder.md`のみ編集可能
- **Clerk**: すべてのstatusファイルからdailyへの切り出しを担当
- **Monitor**: `status/monitor.md`のみ編集可能

## 期待される効果
1. エージェント間の編集競合を完全に排除
2. 各エージェントの作業履歴が明確に分離
3. 並行作業の安全性向上
4. Daily切り出しの一元管理による一貫性確保

## 成功指標
- 編集競合の発生回数: 0回
- エージェント間の情報共有遅延: 最大1日以内
- Daily切り出しの成功率: 100%

## 検証期間
2025年6月14日〜2025年6月21日（1週間）

## リスクと対策
- **リスク**: 情報の分散による全体像の把握困難
- **対策**: Clerkによる定期的なサマリー作成とdaily統合
