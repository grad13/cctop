# H041: インシデント命名規則標準化仮説

**仮説ID**: H041  
**作成日**: 2025年6月15日 nodata  
**作成者**: Clerk Agent  
**検証期間**: 2025年6月15日 - 2025年6月22日  
**ステータス**: 検証中  

## 📋 仮説の概要

インシデントファイルの命名規則を`INC-YYYYMMDD-XXX-title.md`形式に標準化し、検索性・管理性を向上させる。

## 🎯 問題

### 現状の問題
- インシデント番号の一意性が日付に依存
- 同日に複数インシデントが発生した場合の区別が困難
- hypotheses、protocolsとの命名規則の不統一

### 影響
- インシデント検索の困難さ
- 参照時の混乱
- 管理の煩雑さ

## 💡 仮説

**「インシデントファイルに連番を導入することで、一意性を保証し管理を効率化する」**

### 命名規則の詳細

#### フォーマット
```
INC-YYYYMMDD-XXX-title.md
```

#### 各要素の説明
- **INC**: インシデントを示す固定接頭辞
- **YYYYMMDD**: 発生日（例：20250615）
- **XXX**: 3桁の連番（000から開始、日付リセットなし）
- **title**: 簡潔な説明（kebab-case）

#### 例
```
INC-20250615-001-coder-self-resolution.md
INC-20250615-002-monitor-claude-exclusion.md
INC-20250616-003-process-violation.md
```
