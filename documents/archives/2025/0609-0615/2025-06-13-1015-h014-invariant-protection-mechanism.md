# H014: 不変要素保護メカニズム

**仮説ID**: H014  
**作成日時**: 2025年6月13日 10:15  
**カテゴリ**: Meta（セキュリティ・プロセス改善）  
**ステータス**: 実施中  
**緊急度**: 最高（セキュリティインシデント対応）

## 🚨 インシデント詳細

### 発生事象
1. **Dominant/H000への変更試行**: 不変の基盤要素に対して統合・変更を試みた
2. **H011存在確認失敗**: 適切な確認せずに「存在しない」と誤判断
3. **必須プロセス違反**: GUIDELINES.md確認、LS実行をスキップ

### 影響度
- **セキュリティレベル**: 最高（不変要素への不正アクセス試行）
- **信頼性損失**: 基本プロセス無視による重大な信頼性低下
- **潜在的損害**: 基盤システムの破壊リスク

## 🧪 仮説

**「不変要素（Dominant/H000等）への保護メカニズムを明確化し、アクセス前の強制確認プロセスを導入することで、セキュリティインシデントを防止できる」**

## 🎯 解決策

### 1. 不変要素リストの明確化
```
【絶対不変要素】
- dominants/のすべて（DDD0等）
- H000（永続仮説）

【改善対象（不変ではない）】
- GUIDELINES.md → H004で管理・改善
- status/{agent}.md → H039で管理・改善
```

### 2. アクセス前必須確認プロセス
1. **対象確認**: 操作対象が不変要素リストに含まれないか確認
2. **ユーザー承認**: 不変要素の場合、必ずユーザーに確認
3. **理由説明**: なぜその操作が必要かを明確に説明
4. **代替案提示**: 不変要素を変更せずに済む方法を提案

### 3. ファイル存在確認の強制プロセス
```bash
# 必須実行順序
1. ls でディレクトリ確認
2. grep で関連ファイル検索
3. GUIDELINES.md で配置確認
4. それでも見つからない場合のみ「存在しない」と判断
```

## 📏 検証方法

### 成功指標
1. **不変要素への不正アクセス**: 0件

## 💡 即時適用ルール

### 今後の作業における必須事項
1. **「統合」「変更」「修正」の単語を見たら一時停止**
   - 対象が不変要素でないか確認
   - ユーザーに確認を取る

2. **ファイル存在確認は3段階必須**
   - LS → Grep → GUIDELINES.md確認

3. **H000/Dominant言及時は赤信号**
   - 即座に作業停止
   - 「これは不変要素です」と明示
   - ユーザーの明示的指示を待つ
