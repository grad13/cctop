## 🎯 概要

仮説のライフサイクル（作成→検証→評価→昇格/廃棄）を明確化し、期限管理プロセスにより、仮説の有効活用と管理効率を向上させる。

## 📋 仮説ステータス定義

### ステータス一覧
```
1. 【提案中】: 仮説立案済み、検証開始前
2. 【検証中】: 検証期間内でアクティブ
3. 【期限切れ・評価待ち】: 検証期間終了、評価未実施
4. 【成功・昇格待ち】: 成功基準達成、プロトコル昇格待ち
5. 【失敗・アーカイブ待ち】: 成功基準未達、アーカイブ待ち
6. 【永続検証】: P000のような無期限検証
```

### ステータス遷移図
```
提案中 → 検証中 → 期限切れ・評価待ち → 成功・昇格待ち → プロトコル化
                                    → 失敗・アーカイブ待ち → アーカイブ
                 → 永続検証（特別指定のみ）
```

## ⏰ 期限管理プロセス

### 日次チェック（必須）
```
□ protocols/README.mdで期限切れ確認
□ 期限切れ項目に【期限切れ・評価待ち】マーク追加
□ CLAUDE.mdの仮説リスト更新
□ 評価タスクの作成
```

### 期限ルール
- **48時間ルール**: 期限切れから48時間以内に評価実施
- **7日間ルール**: 評価なし7日間で自動アーカイブ扱い
- **通知タイミング**: 期限3日前、1日前、当日
