# REP-0083: P022強化計画 - Dominants参照チェック組み込み

**作成日時**: 2025年6月19日 19:00  
**作成者**: Clerk Agent  
**種別**: プロトコル改善提案  
**優先度**: 高（システム整合性向上）  
**関連**: REP-0080, REP-0082, P022, P007

## 1. 問題認識

### 🚨 **P022の盲点発見**
今回のdominants参照整合性問題（REP-0082）は、**P022「ディレクトリ総合整合性プロトコル」で検出できるべき問題**だったが、実際には検出されなかった。

### 1.1 なぜP022で検出できなかったか

#### **P022の現在の対象範囲**
- ✅ README.md整合性チェック
- ✅ 一般的なbroken links検出
- ✅ プロトコル参照整合性（P007連携）
- ❌ **Dominants内の参照整合性**（明示的対象外）

#### **具体的な盲点**
1. **dominants/**ディレクトリが特別扱いされていない
2. **最高位原則の参照**が通常の文書と同じ扱い
3. **P040不変要素保護**との関係性が整理されていない

## 2. 問題の深刻性

### 2.1 システム整合性への影響
- **最高位原則の信頼性**: Dominants参照エラーは基盤の不安定化
- **P022の実効性**: 重要な問題を見逃すプロトコルの価値低下
- **予防効果の限界**: 事後対応ではなく予防が目的のはず

### 2.2 今回の事例
- **DDD0**: H000/H004への存在しない参照（archive移動済み）
- **DDD2**: P027の状況不明記、P043の不正確な参照
- **検出**: 手動発見（P022では検出されず）

## 3. P022強化提案

### 3.1 基本方針
**「Dominants参照整合性を最高優先度でチェック」**

#### なぜ最高優先度か
1. **影響範囲**: プロジェクト全体の基盤
2. **不変性**: P040で保護されている要素
3. **参照頻度**: 全statusファイル・多数のreportsで参照

### 3.2 具体的強化内容

#### Phase 0: Dominants特別チェック（新設）
```bash
# Phase 0.1: Dominants参照整合性チェック（15分）
echo "=== DOMINANTS REFERENCE INTEGRITY CHECK ==="

# dominants内から参照されているすべてのファイル・プロトコル番号を抽出
grep -o "[PH][0-9][0-9][0-9]" documents/rules/dominants/*.md | sort | uniq

# 各参照の存在確認
for ref in $(grep -o "[PH][0-9][0-9][0-9]" documents/rules/dominants/*.md | sort | uniq); do
    echo "Checking $ref..."
    # Pで始まる場合はprotocolsディレクトリで確認
    # Hで始まる場合はarchive検索も含む
done

# dominants内の具体的ファイル参照も確認
grep -o "documents/[^)]*\.md" documents/rules/dominants/*.md
```

#### Phase 1強化: 優先度システム導入
```bash
# 修正前: 通常のbroken links検出
grep -r "documents/" --include="*.md" documents/

# 修正後: 優先度別broken links検出
echo "=== CRITICAL: Dominants references ==="
grep -r "documents/" documents/rules/dominants/ --include="*.md"

echo "=== HIGH: CLAUDE.md references ==="  
grep -r "documents/" CLAUDE.md

echo "=== MEDIUM: Status files references ==="
grep -r "documents/" documents/agents/status/ --include="*.md"

echo "=== LOW: Other references ==="
# その他の参照
```

### 3.3 チェック強化詳細

#### 新Phase 0: Dominants完全整合性チェック
1. **参照抽出**: dominants内のすべての参照を抽出
2. **存在確認**: 各参照先の存在確認（protocols/archive含む）
3. **状況確認**: 統合済み・廃止済みの状況記載確認
4. **即座修正**: Critical問題の即時対応

#### 既存Phase 3強化: P007との連携強化
- **dominants専用セクション**をP007に追加
- **最高優先度**でdominants参照を処理
- **P040遵守**した修正方針の適用
