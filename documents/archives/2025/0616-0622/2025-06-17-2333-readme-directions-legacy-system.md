# Directions - 確立された方法論

**作成日時**: 2025年6月13日 15:00  
**目的**: 検証済みの仮説から確立された方法論を管理

## 📋 概要

Directionsは、仮説検証を経て有用性が確認された方法論・プロセス・ルールを格納するディレクトリです。CLAUDE.mdのコンテキスト肥大化を防ぎ、必要に応じて選択的に参照することを目的とします。

## 🎯 Directionsとは

### 定義
- **検証済みの仮説**: 一定期間の検証を経て効果が確認された仮説
- **確立された方法論**: プロジェクトの標準的な手法として採用されたプロセス
- **選択的参照ルール**: 状況に応じて必要な時のみ参照される指針

### 仮説からDirectionへの昇格条件
1. **検証期間の完了**: 設定された検証期間を満了
2. **成功指標の達成**: 目標とした指標を達成
3. **実用性の確認**: 実際の開発で有用性を確認
4. **汎用性の確保**: 特定状況に限定されない適用可能性

## 🔄 運用フロー

### 1. 仮説からの昇格
```
hypotheses/hXXX-*.md
    ↓ 検証完了・成功
directions/[category]/dXXX-*.md
    ↓
CLAUDE.mdから参照削除
```

### 2. 参照方法
- **通常時**: CLAUDE.mdのみ参照（directions不要）
- **特定作業時**: 関連するdirectionを明示的に参照
- **問題発生時**: 該当カテゴリのdirectionsを確認

### 3. メンテナンス
- **定期レビュー**: 四半期ごとに有効性確認
- **統廃合**: 類似directionの統合
- **アーカイブ**: 陳腐化したdirectionの移動

## 📝 Direction作成ガイドライン

### 必須要素
1. **適用条件**: いつ参照すべきか
2. **具体的手順**: 実行可能なステップ
3. **期待効果**: 適用による効果
4. **注意事項**: 落とし穴や制約
5. **関連資料**: 元仮説へのリンク

### ファイル命名規則
```
d[番号]-[内容簡潔表現].md
例: d001-git-commit-process.md
```

## ⚠️ 重要な注意事項

### Directionsは追記対象外
一度確立されたDirectionは、原則として内容の追記・修正を行いません。改善が必要な場合は：
1. 新たな仮説として検証
2. 成功後、新しいDirectionとして作成
3. 旧Directionはアーカイブ

### CLAUDE.mdとの関係
- **CLAUDE.md**: コアルール・即座適用が必要なもの
- **Directions**: 状況依存・選択的適用のもの
- **移行基準**: CLAUDE.mdが肥大化した際の切り出し候補

## 🚀 今後の展望

### CLAUDE.md肥大化対策（仮説）
CLAUDE.mdが大きくなりすぎた場合、以下の基準でdirectionsへ移行：
1. 使用頻度が低いルール
2. 特定状況でのみ適用されるルール
3. 詳細な手順説明が必要なルール

この方針は将来的な仮説として記録し、必要に応じて検証する。
