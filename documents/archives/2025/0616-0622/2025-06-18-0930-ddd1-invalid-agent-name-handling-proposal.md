# REP-0065: DDD1更新提案 - 存在しないAgent名への対応

**作成日**: 2025年6月18日 09:30  
**作成者**: Clerk Agent  
**ステータス**: 提案  
**カテゴリー**: Dominant規則更新  
**関連文書**: DDD1（Agent役割必須システム）

## 疑問点・決定事項
- [ ] DDD1への追記場所の承認
- [ ] エラーメッセージの文言承認
- [ ] CLAUDE.mdへの反映方法

---

## 1. 背景

現在、存在しないAgent名を指定された場合の明確な対応が定義されていません。これにより、ユーザーが誤ったAgent名を指定した際に適切なフィードバックが提供されない可能性があります。

## 2. 提案する更新内容

### 2.1 DDD1への追加箇所

**「### 定義された役割（現在5つ）」セクションの直後に以下を追加：**

```markdown
### 有効なAgent名（2025年6月18日現在）
**以下の5つのAgent名のみが有効である**：
1. **Builder Agent**: 機能実装・コード作成・技術実装
2. **Validator Agent**: 品質検証・テスト・デプロイ・品質保証
3. **Architect Agent**: システム設計・技術仕様・ロードマップ管理
4. **Clerk Agent**: 文書管理・プロセス策定・CLAUDE.md管理
5. **Inspector Agent**: 統計監視・データ分析・システム運用監視
```

**「## ✅ 必須実行事項」セクションに以下を追加：**

```markdown
### 4. 無効なAgent名への対応義務
**存在しないAgent名を指定された場合**：
1. 即座に作業停止
2. 以下のメッセージを返答：
   「指定されたAgent名 '{指定されたAgent名}' は存在しません。
   有効なAgent名は以下の5つです：
   - Builder（コード実装）
   - Validator（品質検証）
   - Architect（設計・仕様）
   - Clerk（文書管理）
   - Inspector（監視・分析）
   正しいAgent名を指定してください。」
3. ユーザーの再指定を待機
```

### 2.2 CLAUDE.mdへの追加

**「## 👥 agent権限システム」セクションの「### ⚠️ 権限システムの絶対原則」の後に追加：**

```markdown
### 🚫 無効なAgent指定への対応
存在しないAgent名（例：Coder、Monitor、その他）を指定された場合：
1. **即座停止** - 作業を開始せず停止
2. **エラー通知** - 有効なAgent名リストを提示
3. **再指定待機** - ユーザーの正しい指定を待つ

**有効なAgent**: Builder, Validator, Architect, Clerk, Inspector の5つのみ
```

## 3. 実装手順

1. **ユーザー承認取得**（本レポート）
2. **DDD1更新**（承認後即座）
3. **CLAUDE.md更新**（DDD1更新と同時）
4. **更新履歴記録**

## 4. 期待効果

1. **明確なエラーハンドリング**: 誤ったAgent名への即座の対応
2. **ユーザビリティ向上**: 正しいAgent名の即座の提示
3. **システム整合性**: 廃止されたAgentへのアクセス防止

## 5. リスクと対策

- **リスク**: 既存のワークフローへの影響
- **対策**: エラーメッセージに正しいAgent名を明示することで混乱を最小化
