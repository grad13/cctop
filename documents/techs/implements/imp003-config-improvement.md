# Configuration System Specification Improvement Plan

**作成日**: 2025-06-22 10:45  
**作成者**: Architect Agent  
**対象**: cctop設定システム仕様書の改善計画  
**現状調査**: 実装コードとテストから仕様を再定義

## 📋 計画概要

本計画は、設定システムの実装と仕様書のギャップを解消し、より正確で実用的な仕様書を策定するための改善計画です。

## 🔍 現状分析結果

### 1. 実装と仕様の主要な差異

#### モジュール構成の相違
| 仕様書記載 | 実装状況 | 差異の理由 |
|-----------|---------|-----------|
| 8モジュール想定 | 4モジュール実装 | 機能統合による効率化 |
| config-validator.js | defaults.jsに統合 | 検証とデフォルト定義の密結合 |
| environment-parser.js | config-manager.jsに統合 | 階層的読み込みの一部として実装 |
| cli-parser.js | config-manager.jsに統合 | 同上 |
| config-watcher.js | 未実装 | Phase 3として後回し |

#### 追加実装された機能（仕様書未記載）
1. **JSファイル設定サポート** - module.exportsによる動的設定
2. **JSONコメント除去** - strip-json-commentsライブラリ使用
3. **高度なマージ戦略** - 6種類のarrayマージ戦略
4. **カスタムマージルール** - アプリケーション固有のマージロジック定義可能
5. **コンフリクト追跡** - マージ時の設定衝突を記録・レポート

### 2. 統合レベルの問題

#### t001評価で判明した問題
- **キャッシュ設定統合**: ConfigManagerの設定がCacheManagerに反映されない
- **プロファイル適用順序**: 環境変数がプロファイル設定を上書きする問題
- **CLI引数処理**: bin/cctopレベルでの統合が不完全

## 🎯 改善計画

### Phase 1: 仕様書の現実反映（即時実施）

#### 1.1 モジュール構成の再定義

```markdown
## 実装モジュール構成（実装済み）

### コアモジュール（4ファイル）

- config-loader.js: ファイル読み込みと解析

### 将来実装予定モジュール

```
