# バグ記録テンプレート

**このテンプレートをコピーして、新しいバグ記録ファイルを作成してください**

## ファイル命名規則
```
documents/bugs/[YYYYMMDD]-[簡潔なタイトル].md
例: documents/bugs/20250614-timebox-pause-timer-continues.md
```

## テンプレート内容

```markdown

**担当**: Builder Agent（実装）/ Validator Agent（検証）  
**関連コンポーネント**: [関連する機能・ファイル名]

## 使用方法

1. **即座記録**: バグ発見時、このテンプレートをコピーして新ファイル作成
2. **1ファイル1バグ**: 1つのバグにつき必ず1つのファイルを作成（**絶対制約**）
3. **継続更新**: 調査・修正の進展に応じて内容を更新
4. **完了確認**: 解決時にステータスを「解決済み」に更新
5. **品質チェック**: コミット前に記録内容の完全性を確認

## 重要度の判定基準

- **高**: システム停止、データ損失、セキュリティ問題
- **中**: 機能不全、使用性に大きな影響
- **低**: 軽微な表示問題、非クリティカルな機能

## pre-commitチェック対象キーワード

以下のキーワードがコミットメッセージに含まれる場合、バグ記録が必須：
- bug, バグ
- fix, 修正
- error, エラー  
- issue, 問題
- crash, クラッシュ
- broken, 動作不良
