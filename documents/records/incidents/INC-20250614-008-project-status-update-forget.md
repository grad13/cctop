# インシデント: LEGACY_STATUS.md更新忘れ（再発）

**インシデントID**: INC-20250614-004  
**発生日時**: 2025年6月14日 04:45  
**重要度**: 高  
**カテゴリ**: プロセス違反  
**関連インシデント**: H017違反、H004類似パターン

## 🔴 問題の詳細

### 発生した事象
1. **包括的デバッグシステム実装後、LEGACY_STATUS.md更新忘れ**
   - H020準拠のデバッグシステムを実装
   - Git commit/pushまで実行
   - LEGACY_STATUS.mdを更新せずに進行
   
2. **ユーザーからの指摘で発覚**
   - 「LEGACY_STATUS.mdを更新してないですよね」
   - 「インシデントです」

## 💥 影響

### プロセス的影響
- H017（プロジェクト状況把握プロセス）違反
- 作業進捗の追跡性が失われる
- 次回セッション時の状況把握が困難

### パターン認識
- H004（documents更新忘れ）と同じパターン
- 実装に集中して記録を忘れる傾向

## 🔍 原因分析

### 直接原因
1. **実装完了後の興奮**
   - デプロイまで完了して安心
   - 次のステップ（デバッグ実行）に意識が向いた

2. **チェックリスト不在**
   - Git操作後の必須タスクリストがない
   - 記憶に依存した運用

### 根本原因
- **タスク完了の定義が不明確**
- **実装とドキュメントを別タスクと認識**

## 🛠️ 改善策

### 即時対応
LEGACY_STATUS.mdを即座に更新（実施済み）

### プロセス改善
1. **Git操作チェックリスト作成**
   ```
   □ コード変更完了
   □ LEGACY_STATUS.md更新
   □ git add -A
   □ git commit
   □ git push
   ```

2. **完了の定義明確化**
   - コード実装 + ドキュメント更新 = 1つのタスク
   - 片方だけでは未完了

### 技術的対策（実装済み）
1. **`.git-commit-checklist`作成**
   - 必須確認事項の明記
   - LEGACY_STATUS.md更新項目の詳細

2. **`scripts/git-commit-with-check.sh`作成**
   - LEGACY_STATUS.md変更の自動確認
   - 未更新時のコミット拒否
   - チェックリスト表示と確認

3. **H023仮説策定**
   - LEGACY_STATUS.md更新強制メカニズム
   - 技術的強制とプロセス改善

## 📝 教訓

**「実装完了」≠「タスク完了」**

実装が終わっても、それを記録するまでがタスクである。特にLEGACY_STATUS.mdは継続性の要であり、更新は必須。
