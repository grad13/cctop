# インシデント対応チェックリスト不参照問題

**インシデントID**: INC-20250614-014  
**発生日時**: 2025年6月14日 14:18  
**報告者**: ユーザー  
**対応者**: Clerk Agent  

## 📋 概要

エージェントがインシデント対応時にチェックリストを参照せず、記憶に頼って対応している。これにより対応品質にばらつきが生じ、プロセスが形骸化している。

## Phase 2: 詳細分析

### 現象の詳細
- **何が起きたか**: インシデント対応時にチェックリストを参照していない
- **期待動作**: 必ず`meta/checklists/incident-response-checklist.md`を参照して5フェーズ対応
- **実際の動作**: 記憶に基づく対応、チェックリスト参照なし
- **影響範囲**: 全エージェント、全インシデント対応品質

### 原因分析（5 Whys法）

**Why 1**: なぜチェックリストを参照しないか？
→ チェックリストの存在を知っているため、参照せずとも対応できると判断している

**Why 2**: なぜ参照せずに対応できると判断するか？
→ 過去の対応経験から、大まかな流れを記憶している

**Why 3**: なぜ記憶に頼る対応をするか？
→ チェックリスト参照が「推奨」であり「強制」でないと認識している

**Why 4**: なぜ強制でないと認識するか？
→ CLAUDE.mdにチェックリスト参照の強制明記がない

**Why 5**: なぜ強制明記がないか？
→ **根本原因**: インシデント対応プロセスの強制力が不足している

## Phase 3: 対策立案

### 即時対応
1. 本インシデント対応自体をチェックリスト完全準拠で実施（実証）
2. CLAUDE.mdへの強制ルール追加

### 再発防止策
1. **CLAUDE.md強化**
   - インシデント発生時のチェックリスト参照を絶対必須化
   - 参照しない場合は二次インシデント扱い

2. **チェックリスト自体の改善**
   - 冒頭に「必ず毎回参照」の強調追加
   - Phase毎のチェックボックス活用必須化

3. **H031仮説立案**
   - チェックリスト強制参照システムの検証
   - **注記**: H031は現在P031（プロセス遵守強制プロトコル）としてprotocolsに統合されている
