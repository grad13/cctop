# INC-20250618-002: documents/handoffsディレクトリ再作成問題

## Phase 1: 初期記録（即座）

**発生日時**: 2025年6月18日 21:35  
**発見者**: Clerk Agent  
**影響範囲**: アーキテクチャ一貫性・REP-0070成果の毀損  
**緊急度**: High - アーキテクチャ重複による混乱リスク  

**問題概要**:
REP-0070で統合・削除したはずのdocuments/handoffsディレクトリが再作成されている。ワークスペースrootのhandoffs/は正常に存在・機能しているが、documents/handoffs/が空ディレクトリとして再出現し、アーキテクチャの一貫性を損なっている。

## Phase 2: 詳細分析（5 Whys）

### 調査結果
1. **Git履歴確認**:
   - コミット`fe84abe`（18:14）: documents/handoffsを作成
   - コミット`0853bc2`（18:37）: `git rm -r documents/handoffs/`で削除
   - 削除は成功しているが、空ディレクトリがローカルに残存

2. **現在の状態**:
   - Git上では削除済み（tracked filesから除外）
   - ローカルファイルシステムに空ディレクトリとして残存
   - ユーザーにより手動削除済み（21:35頃）

### 5 Whys分析
1. **なぜdocuments/handoffsが再作成されたか？**
   → `git rm -r`実行後も物理的な空ディレクトリが残存したため

2. **なぜ空ディレクトリが残存したか？**
   → Gitは空ディレクトリを追跡しないため、`git rm`では物理削除されない

3. **なぜ空ディレクトリの残存に気づかなかったか？**
   → REP-0070作業完了後の物理的確認が不足していた

4. **なぜ物理的確認が不足したか？**
   → Git操作だけで完全削除されると想定していた

5. **なぜこの想定が誤っていたか？**
   → Gitの仕様（空ディレクトリの非追跡）に関する理解不足

### 根本原因
**Gitの空ディレクトリ非追跡仕様とディレクトリ統合作業時の物理的確認不足**

## Phase 3: 対策立案

### 即時対応

2. **参照更新確認**: 全ファイルでdocuments/handoffsへの参照がないことを確認
3. **P016・REP-0022更新**: handoffsがワークスペースrootのみであることを再強調

### 再発防止策
