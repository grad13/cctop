# インシデント: 完了作業のdaily切り出し忘れ

**インシデントID**: INC-20250614-005  
**発生日時**: 2025年6月14日 05:40  
**重要度**: 高  
**カテゴリ**: プロセス違反  
**関連**: CLAUDE.md記載のドキュメント管理ポリシー違反

## 🔴 問題の詳細

### 発生した事象
1. **Quick Switch問題が解決完了したが、daily切り出しを忘れた**
   - 12時間以上のデバッグ作業が完了
   - LEGACY_STATUS.mdに「解決済み」と記載
   - しかし、詳細をdailyに切り出していない

2. **ユーザーからの指摘で発覚**
   - 「終わった箇所はdailyに切り取りして保存ですよね」
   - 明確なルール違反

## 💥 影響

### プロセス的影響
- 作業記録の完全性が損なわれる
- 後から振り返る際の詳細情報が失われる
- CLAUDE.mdのドキュメント管理ポリシー違反

### パターン認識
- 問題解決の達成感で次のプロセスを忘れる
- LEGACY_STATUS.md更新で満足してしまう

## 🔍 原因分析

### 直接原因
1. **問題解決の興奮**
   - 12時間以上の難問が解決した達成感
   - 「解決した！」で思考が止まった

2. **プロセスの不完全な理解**
   - LEGACY_STATUS.md更新 = 完了と誤認識
   - daily切り出しまでが1セットという認識不足

### 根本原因
- **タスク完了の定義が不明確**
- **チェックリストにdaily切り出しが含まれていない**

## 🛠️ 改善策

### 即時対応（今すぐ実施）
1. Quick Switch問題解決の詳細をdailyに切り出す
2. LEGACY_STATUS.mdから該当部分を整理

### プロセス改善
1. **.git-commit-checklistに追加**
   ```
   - [ ] 完了した作業をdailyに切り出した（該当する場合）
   ```

2. **タスク完了の定義を更新**
   - コード実装
   - LEGACY_STATUS.md更新
   - **dailyへの切り出し** ← 追加
   - git commit

3. **H024仮説：daily切り出し自動リマインダー**
   - 「解決完了」「完了」等のキーワードでトリガー
   - daily切り出しを促すプロンプト

## 📝 教訓

**「完了」には必ず「記録の移動」が含まれる**

LEGACY_STATUS.mdは現在進行形の作業用。完了したものは必ずdailyに移動して、詳細な記録として保存する。
