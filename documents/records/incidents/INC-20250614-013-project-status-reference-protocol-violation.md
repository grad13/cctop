# インシデント記録: LEGACY_STATUS.md参照プロトコル違反

**インシデントID**: INC-20250614-012  
**発生日時**: 2025年6月14日 13:35  
**報告者**: User  
**対応者**: Coder  
**重要度**: Critical  

## 概要

Coderが廃止済みの`documents/LEGACY_STATUS.md`ファイルを参照しようとし、記録管理システム移行の認識漏れが発覚。H025即時記録プロトコル違反の根本的問題として判明。

## 現象の詳細

### 期待動作
- 作業記録は`documents/agents/status/coder.md`で管理される
- H025プロトコルに従い即座に記録更新を実行する
- 廃止されたファイル参照は一切行わない

### 実際の動作
- `documents/LEGACY_STATUS.md`へのRead操作実行
- "File does not exist"エラー発生
- 廃止済みファイル参照を継続
- 作業記録更新の長時間放置

### 影響範囲
- **Coder記録管理**: 完全機能停止
- **H025プロトコル遵守**: 全面違反
- **プロジェクト状況把握**: 重大な阻害
- **インシデント対応品質**: 表面的対応のみ

## 原因分析（5 Whys法）

**Why 1**: なぜLEGACY_STATUS.mdを参照したか？
→ ユーザーに記録更新を指摘され、従来の参照パターンで対応した

**Why 2**: なぜ従来パターンを使ったか？
→ 記録管理システム移行（LEGACY_STATUS.md → status/coder.md）を認識していなかった

**Why 3**: なぜ移行を認識していなかったか？
→ CLAUDE.mdの情報更新を把握せず、古い認識のままだった

**Why 4**: なぜCLAUDE.md更新を把握していなかったか？
→ プロジェクト開始時のセッション情報確認が不完全だった

**Why 5**: なぜセッション開始確認が不完全だったか？
→ **根本原因**: セッション継続時の状況把握プロセスが機能していない

## 対策

### 即時対応 ✅
1. status/coder.md更新完了
2. インシデント記録作成（本ファイル）

### 再発防止策
1. **セッション継続時確認プロセス強化**
   - CLAUDE.md読み直し必須
   - 前回セッション状況の把握必須
   - 管理ファイル変更の確認必須

2. **記録管理システム統一化**
   - 全エージェントでstatus/{agent}.md形式統一
   - 古いLEGACY_STATUS.md参照の完全削除

3. **H025プロトコル強化**
   - 作業開始と同時に記録更新
   - 30分ごとの強制記録確認

## 実装予定

### CLAUDE.md更新
- セッション継続時の必須確認プロセス追加
- 古いLEGACY_STATUS.md参照の削除

### 仮説立案
- H029: セッション継続時状況把握プロセス
- H030: 記録管理システム統一化

3. **プロトコル違反の連鎖**: 一つの認識漏れが複数の重大違反を引き起こす
