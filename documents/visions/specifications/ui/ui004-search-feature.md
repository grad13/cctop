# CCTop 検索機能設計仕様

**作成日**: 2025-06-21  
**作成者**: Inspector Agent  
**目的**: htopリスペクトのvim風検索機能の詳細設計

## 🎯 設計方針

### 基本コンセプト
- htop/vimライクな検索体験
- シンプルで直感的な操作
- リアルタイムフィルタリング
- 最小限の画面遷移

### MVP方針
- 単純な文字列マッチのみ（正規表現は将来拡張）
- 大文字小文字を区別しない
- 部分一致検索
- 検索状態は永続化しない（ESCで即解除）

## 🔍 検索動作仕様

### 検索開始
1. `/`キー押下で検索モード開始
2. 画面最下部に検索バー表示
3. 既存のヘルプ行を検索バーで置換
4. カーソルを検索バーに表示

### 検索中の動作
```
動作フロー:
1. 文字入力 → 即座にフィルタリング
2. バックスペース → 文字削除・再フィルタ
3. 空文字列 → 全件表示
4. ESC → 検索キャンセル・通常表示に戻る
```

### マッチング仕様
- **検索対象**: ファイルパス全体（directory + file_name）
- **マッチ方式**: 大文字小文字を区別しない部分一致
- **例**: 
  - 検索文字列 "api" は以下にマッチ:
    - `src/api/user.js`
    - `lib/api-client.js`
    - `API_CONSTANTS.js`

## 🖥️ 画面表示仕様

### 通常時の画面（最下部）
```
─────────────────────────────────────────────────────────────────────────────────
Stream: All  Filters: Create:ON Modify:ON  Items: 20/234
[a] All  [u] Unique  [s,c,m,v,d] Filters  [/] Search  [q] Quit
```

### 検索時の画面（最下部）
```
─────────────────────────────────────────────────────────────────────────────────
Stream: All  Searching  Matches: 5/234                    ← ステータス変更
[ESC] Cancel Search  [Enter] Keep Filter                  ← 操作ヘルプ変更
/api▌                                                     ← 検索バー
```

### 表示要素の詳細
| 要素 | 説明 |
|------|------|
| `Searching` | 検索モード中であることを示す |
| `Matches: 5/234` | マッチ数/全体数 |
| `/api▌` | 検索バー（/始まり、カーソル表示） |

## ⌨️ キーボード操作

### 検索モード中の操作
| キー | 動作 | 説明 |
|------|------|------|
| 文字キー | 文字入力 | 検索文字列に追加 |
| `Backspace` | 文字削除 | 最後の1文字を削除 |
| `Ctrl+U` | 全削除 | 検索文字列をクリア |
| `ESC` | キャンセル | 検索終了・通常表示 |
| `Enter` | （将来拡張） | 検索維持など |

### 検索モード中に無効化されるキー
- `a`, `u`: モード切替
- `s`, `c`, `m`, `v`, `d`: イベントフィルタ
- `↑`, `↓`: フォーカス移動（検索結果内では有効）
- その他のコマンドキー

## 🔄 状態管理

### 検索状態
```javascript
searchState = {
  isSearching: false,      // 検索モード中か
  searchQuery: '',         // 検索文字列
  matchedCount: 0,         // マッチ件数
  filteredActivities: []   // フィルタ結果
}
```

### 状態遷移
```
通常モード → [/キー] → 検索モード → [ESC] → 通常モード
                         ↓
                    [文字入力] → リアルタイムフィルタ
```

## 💻 実装ガイドライン

### モジュール構成
- `SearchHandler`: 検索モード管理・キー入力処理
- `SearchFilter`: フィルタリングロジック
- `SearchRenderer`: 検索バー表示

### フィルタリング実装
```javascript
// 簡易実装例
function filterActivities(activities, query) {
  if (!query) return activities;
  
  const lowerQuery = query.toLowerCase();
  return activities.filter(activity => {
    const fullPath = `${activity.directory}${activity.file_name}`.toLowerCase();
    return fullPath.includes(lowerQuery);
  });
}
```

### レンダリング考慮事項
- 検索バーは既存のレイアウトを崩さない
- カーソル表示は点滅させる（オプション）
- 日本語入力は考慮しない（MVP）
