# CCTop レンダリング・更新仕様

**作成日**: 2025-06-21  
**作成者**: Inspector Agent  
**目的**: CLI UIの画面更新とレンダリング処理の詳細仕様

## 🔄 更新システム概要

### 基本アーキテクチャ
- **イベント駆動更新**: ファイル変更イベントをトリガーに画面更新
- **バッファリング描画**: ちらつき防止のための二重バッファ方式
- **非同期処理**: ファイル監視と画面描画の独立動作

## 📊 リアルタイム更新

### 更新フロー
```
ファイル変更検出
    ↓
イベント処理（event-processor）
    ↓
データベース更新
    ↓
表示データ取得
    ↓
画面レンダリング
```

### 更新タイミング
- **即時更新**: ファイル変更検出後、即座に画面更新開始
- **バッチ処理**: 複数イベントが短時間に発生した場合はまとめて処理
- **設定可能な間隔**: デフォルトは即時、設定により調整可能

## 🖼️ バッファリングレンダリング

### BufferedRendererクラス
```javascript
class BufferedRenderer {
  constructor() {
    this.buffer = [];        // 描画バッファ
    this.cursorSaved = false; // カーソル位置保存フラグ
  }
  
  // バッファクリア
  clear() { /* ... */ }
  
  // 行追加
  addLine(line) { /* ... */ }
  
  // バッファ内容を画面に描画
  render() { /* ... */ }
}
```

### 主要機能
1. **二重バッファリング**
   - メモリ上で完全な画面を構築
   - 一度に画面全体を更新

2. **カーソル制御**
   - 描画前にカーソル位置を保存
   - 描画後に元の位置に復元

3. **エスケープシーケンス活用**
   - ANSI エスケープコードで効率的な画面制御
   - 部分更新による高速化

## ⚡ パフォーマンス最適化

### デバウンス処理
```javascript
// 16ms（約60fps）のデバウンス
renderDebounceTimer = null;
renderDebounceDelay = 16;

displayWithFocusDebounced() {
  if (this.renderDebounceTimer) {
    clearTimeout(this.renderDebounceTimer);
  }
  
  this.renderDebounceTimer = setTimeout(() => {
    this.displayWithFocus();
  }, this.renderDebounceDelay);
}
```

### 最適化手法
1. **差分更新の考慮**
   - 将来的に変更部分のみ更新する差分アルゴリズム導入可能

2. **レンダリング制限**
   - 人間の知覚限界を考慮した60fps制限
   - CPU使用率の抑制

3. **バッチ処理**
   - 短時間の連続イベントをまとめて処理
   - データベースアクセスの削減

## 🎯 表示の安定化

### 画面サイズ管理
- **固定行数表示**: displayLimit設定に基づく固定行数
- **空行埋め**: 表示データが少ない場合は空行で埋める
- **スクロール制御**: フォーカスモード時の適切なスクロール位置管理

### ちらつき防止策
1. **画面クリアの最小化**
   - 必要な部分のみクリア
   - 全画面クリアは最小限に

2. **同期的な描画**
   - すべての要素を準備してから一度に描画
   - 部分的な更新による視覚的な乱れを防止

3. **ターミナル互換性**
   - 様々なターミナルエミュレータでの動作確認
   - 基本的なANSIエスケープシーケンスのみ使用

## 🔧 実装ガイドライン

### モジュール責務
| モジュール | 責務 |
|-----------|------|
| StreamRenderer | 画面描画の実行・フォーマット |
| BufferedRenderer | バッファ管理・低レベル描画 |
| StreamController | 更新タイミング制御 |
| StreamDisplay | 全体の調整・統合 |

### 状態管理
```javascript
// レンダリング状態
renderingState = {
  isBuffering: false,     // バッファリングモード
  isDirty: false,         // 再描画が必要か
  lastRenderTime: null,   // 最後の描画時刻
  frameCount: 0           // フレームカウント（デバッグ用）
}
```

## 📈 パフォーマンス目標

### 数値目標
- **レンダリング遅延**: 50ms以内
- **フレームレート**: 30-60fps
- **CPU使用率**: アイドル時1%以下
- **メモリ使用量**: バッファサイズ最大1MB

### 測定方法
- レンダリング時間の計測
- フレームスキップの検出
- リソース使用量のモニタリング
