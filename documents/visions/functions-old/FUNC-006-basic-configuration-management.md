## 概要
cctop v0.1.0.0のシンプル設定管理システム。BP-000で実装される基本版。

## 1. 設定ファイルの種類と優先順位

### 1.1 設定ソース（優先順位順）
1. **コマンドライン引数**
   - `--config <path>` で指定されたファイル
   - 最高優先度

2. **ユーザー設定ファイル**
   - パス: `~/.cctop/config.json`
   - インストール時に自動作成
   - コマンドライン引数がない場合のデフォルト

### 1.2 v0.1.0.0では未対応（将来拡張）
- **環境変数**: FUNC-003（高度版）で将来対応
- **プロファイル**: FUNC-003（高度版）で将来対応
- **動的更新**: v0.2.0.0以降で対応予定

## 2. 起動時の設定読み込みフロー

### 2.1 基本フロー
```
1. コマンドライン引数の確認
   ├─ --configが指定されている場合
   │   └─ 指定されたファイルを読み込む
   └─ --configが指定されていない場合
       └─ ~/.cctop/config.jsonを読み込む
```

### 2.2 設定ファイルが存在しない場合の動作
1. **エラー通知**
   ```
   エラー: 設定ファイルが見つかりません: ~/.cctop/config.json
   ```

2. **設定ファイル作成**
   - デフォルト設定で`~/.cctop/config.json`を自動作成
   - 作成成功メッセージを表示
   ```
   設定ファイルを作成しました: ~/.cctop/config.json
   設定を編集してから再度実行してください。
   ```

3. **ユーザー確認待機**
   - 「Enterキーを押して終了...」プロンプトを表示
   - ユーザーがEnterキーを押すまで待機

4. **プログラム終了**
   - exit code 1で終了

## 3. 設定ファイルフォーマット

### 3.1 必須フィールド
```json
{
  "watchPaths": ["./"],
  "ignorePaths": [],
  "display": {
    "maxEvents": 10,
    "refreshInterval": 100
  },
  "database": {
    "path": "~/.cctop/events.db",
    "maxEvents": 10000,
    "cleanupInterval": 3600000
  }
}
```

### 3.2 検証ルール
- 必須フィールドが欠けている場合はエラー
- 型が不正な場合はエラー
- 不明なフィールドは警告を出して無視

## 4. エラーハンドリング

### 4.1 読み込みエラー
- JSONパースエラー: 詳細なエラー位置を表示
- ファイル権限エラー: 権限の確認を促す
- 設定検証エラー: 不正なフィールドを特定して表示

### 4.2 作成エラー
- ディレクトリ作成失敗: 手動作成を案内
- ファイル書き込み失敗: 権限確認を案内
