# Visions - プロジェクトビジョン実現ディレクトリ

**作成日**: 2025年6月19日  
**最終更新**: 2025年6月27日  
**管理者**: Architect Agent  
**目的**: cctopプロジェクトのビジョン実現に関する全体管理

## 📋 概要

visionsディレクトリは、cctopプロジェクトの「作るもの」「作り方」「進捗状況」を統合管理します。3層構造により、設計図をクリーンに保ちながら、部品仕様と実装進捗を適切に分離します。

## 📁 ディレクトリ構成

```
visions/
├── README.md                # このファイル（全体概要）
├── blueprints/             # 設計図（バージョン目標別）
├── code-guides/            # 実装ガイド（コード例・実装パターン）
├── functions/              # Active機能仕様（実装対象確定）
├── pilots/                 # Draft機能実験（パイロット機能）
├── diagrams/               # システム図・アーキテクチャ図
└── versions.md             # バージョン履歴・リリースノート
```

## 🎯 ディレクトリ構造の役割

### blueprints/ - 設計図
- **内容**: バージョン目標ごとの統合設計図
- **例**: v0.1.0.0実装計画、v0.2.0.0拡張計画
- **特徴**: FUNCTIONSを組み合わせた全体設計
- **更新**: 実装計画の策定・改訂時

### code-guides/ - 実装ガイド
- **内容**: 具体的な実装例・コードパターン
- **例**: Event Processor実装、Config Manager実装
- **特徴**: Blueprintの「何を作るか」に対する「どう作るか」
- **更新**: 新しい実装パターン確立時

### functions/ - Active機能仕様
- **内容**: 実装対象として確定したActive機能の仕様
- **例**: FUNC-000（DB基盤）、FUNC-001（ライフサイクル追跡）
- **特徴**: Builder/Validatorの作業対象、番号管理された確定仕様
- **更新**: 機能仕様の修正・改善時

### pilots/ - Draft機能実験
- **内容**: 実験段階のDraft機能、パイロット実装
- **例**: FUNC-900（色カスタマイズ）、FUNC-903（選択モード）
- **特徴**: 自由な実験・大幅変更可能、実装対象外
- **更新**: 新アイデア検証・機能実験時

### diagrams/ - システム図
- **内容**: アーキテクチャ図・状態遷移図
- **例**: ファイルライフサイクル状態図
- **特徴**: 視覚的な理解を助ける図表

## 🔄 バージョン管理方針

### blueprints/の管理原則
- **1バージョン1ファイル**: v0.1.0.0、v0.2.0.0等の目標別
- **命名規則**: `BP-XXX-for-versionYYYY-description.md`
- **一意性確保**: 同じバージョンについて複数の設計図は作成しない
- **クリーン保持**: 進捗情報は記載せず、設計図として純粋に保つ

### バージョン進行フロー
```
v0.1.0.0設計 → 実装・進捗記録 → 完了
                ↓
v0.2.0.0設計 → 実装・進捗記録 → 完了
                ↓
       （継続的な発展）
```

## 📊 文書配置原則

### 新しい文書の配置判定
1. **機能仕様** → `functions/`
   - FUNC-XXX形式で番号管理
   - 個別機能の詳細仕様
2. **バージョン実装計画** → `blueprints/`
   - BP-XXX形式で管理
   - v0.X.Y.Z の全体設計・実装計画
3. **実装コード例** → `code-guides/`
   - CG-XXX形式で管理
   - 具体的な実装パターン・コード例

### 重複防止原則
- **1トピック1ファイル**: 同じ内容の文書は作成しない
- **最新優先**: 古い文書は削除またはGit履歴で管理
- **参照関係**: blueprints → functions → code-guides の階層参照

### 最新性維持原則
- **現在有効な情報のみ**: visions/内は常に最新の仕様・設計・実装ガイドのみを配置
- **歴史的資料の除外**: 廃止された設計や過去バージョンは配置しない
  - 例外: 移行作業中の参照文書（移行完了後は削除）
- **Git履歴での管理**: 削除された文書はGit履歴で参照可能
- **定期的なクリーンアップ**: 月次で文書の有効性を確認し、適時更新・削除

詳細は[P047: visions/ディレクトリ構造分類ガイドライン](../rules/meta/protocols/p047-visions-structure-classification-guide.md)を参照

## 🏗️ Architect権限範囲

### 完全管理権限
- ✅ `blueprints/` - 設計図・実装計画の作成・修正
