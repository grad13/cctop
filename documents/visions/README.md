# Visions - プロジェクトビジョン実現ディレクトリ

**作成日**: 2025年6月19日  
**最終更新**: 2025年6月24日  
**管理者**: Architect Agent  
**目的**: cctopプロジェクトのビジョン実現に関する全体管理

## 📋 概要

visionsディレクトリは、cctopプロジェクトの「作るもの」「作り方」「進捗状況」を統合管理します。3層構造により、設計図をクリーンに保ちながら、部品仕様と実装進捗を適切に分離します。

## 📁 ディレクトリ構成

```
visions/
├── README.md                # このファイル（全体概要）
├── specifications/          # 部品カタログ（詳細仕様）
├── blueprints/             # 設計図（バージョン目標別）
├── functions/              # 最新機能カタログ（実装済み機能）
├── progress/               # 実装進捗（設計図実行状況）
└── versions.md             # バージョン履歴・リリースノート
```

## 🎯 4層構造の設計思想

### specifications/ - 部品カタログ
- **内容**: 個別技術仕様・コンポーネント詳細
- **例**: データベーススキーマ、UI仕様、設定システム
- **特徴**: 再利用可能な技術部品の詳細定義
- **更新**: 技術仕様の変更時（慎重）

### blueprints/ - 設計図
- **内容**: バージョン目標ごとの統合設計図
- **例**: v0.1.0.0実装計画、v0.2.0.0拡張計画
- **特徴**: specificationsの部品を組み合わせた全体設計
- **更新**: 実装中の改善・修正（積極的）

### functions/ - 最新機能カタログ
- **内容**: 実装済み・検証済みの機能管理
- **例**: ファイル監視機能、CLI表示システム、設定管理
- **特徴**: 完成した機能の実績記録・状況追跡
- **更新**: 機能完了・改善時の記録

### progress/ - 実装進捗
- **内容**: 設計図の実行状況・進捗記録
- **例**: Phase完了状況、実装中の課題、実作業ログ
- **特徴**: 設計図とは完全分離、実装過程の記録
- **更新**: 日次・作業毎の進捗記録

## 🔄 バージョン管理方針

### blueprints/の管理原則
- **1バージョン1ファイル**: v0.1.0.0、v0.2.0.0等の目標別
- **命名規則**: `BP-XXX-for-versionYYYY-description.md`
- **一意性確保**: 同じバージョンについて複数の設計図は作成しない
- **クリーン保持**: 進捗情報は記載せず、設計図として純粋に保つ

### バージョン進行フロー
```
v0.1.0.0設計 → 実装・進捗記録 → 完了
                ↓
v0.2.0.0設計 → 実装・進捗記録 → 完了
                ↓
       （継続的な発展）
```

## 📊 文書配置原則

### 新しい文書の配置判定
1. **個別技術仕様** → `specifications/`
   - データベース仕様、API設計、UI仕様等
2. **バージョン実装計画** → `blueprints/`
   - v0.X.Y.Z の全体設計・実装計画
3. **実装作業記録** → `progress/`
   - Phase完了、課題、実作業ログ

### 重複防止原則
- **1トピック1ファイル**: 同じ内容の文書は作成しない
- **最新優先**: 古い文書は即座にarchives/へ移動
- **参照関係**: blueprints → specifications の一方向参照
