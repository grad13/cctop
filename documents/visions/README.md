# Visions - プロジェクトビジョン実現ディレクトリ

**作成日**: 2025年6月19日  
**最終更新**: 2025年6月26日  
**管理者**: Architect Agent  
**目的**: cctopプロジェクトのビジョン実現に関する全体管理

## 📋 概要

visionsディレクトリは、cctopプロジェクトの「作るもの」「作り方」「進捗状況」を統合管理します。3層構造により、設計図をクリーンに保ちながら、部品仕様と実装進捗を適切に分離します。

## 📁 ディレクトリ構成

```
visions/
├── README.md                # このファイル（全体概要）
├── blueprints/             # 設計図（バージョン目標別）
├── code-guides/            # 実装ガイド（コード例・実装パターン）
├── functions/              # 最新機能カタログ（実装済み機能）
├── legacy/                 # 旧specifications（アーカイブ済み）
├── diagrams/               # システム図・アーキテクチャ図
├── tmp/                    # 一時作業用（整理予定）
└── versions.md             # バージョン履歴・リリースノート
```

## 🎯 ディレクトリ構造の役割

### blueprints/ - 設計図
- **内容**: バージョン目標ごとの統合設計図
- **例**: v0.1.0.0実装計画、v0.2.0.0拡張計画
- **特徴**: FUNCTIONSを組み合わせた全体設計
- **更新**: 実装計画の策定・改訂時

### code-guides/ - 実装ガイド
- **内容**: 具体的な実装例・コードパターン
- **例**: Event Processor実装、Config Manager実装
- **特徴**: Blueprintの「何を作るか」に対する「どう作るか」
- **更新**: 新しい実装パターン確立時

### functions/ - 機能仕様カタログ
- **内容**: 個別機能の詳細仕様
- **例**: FUNC-000（DB基盤）、FUNC-001（ライフサイクル追跡）
- **特徴**: 番号管理された機能単位の仕様書
- **更新**: 機能追加・仕様変更時

### legacy/ - 旧specifications（アーカイブ）
- **内容**: 過去のspecifications文書
- **特徴**: 参照用として保持、新規作成は行わない
- **理由**: FUNCTIONSベースの新体系に移行済み

### diagrams/ - システム図
- **内容**: アーキテクチャ図・状態遷移図
- **例**: ファイルライフサイクル状態図
- **特徴**: 視覚的な理解を助ける図表

## 🔄 バージョン管理方針

### blueprints/の管理原則
- **1バージョン1ファイル**: v0.1.0.0、v0.2.0.0等の目標別
- **命名規則**: `BP-XXX-for-versionYYYY-description.md`
- **一意性確保**: 同じバージョンについて複数の設計図は作成しない
- **クリーン保持**: 進捗情報は記載せず、設計図として純粋に保つ

### バージョン進行フロー
```
v0.1.0.0設計 → 実装・進捗記録 → 完了
                ↓
v0.2.0.0設計 → 実装・進捗記録 → 完了
                ↓
       （継続的な発展）
```

## 📊 文書配置原則

### 新しい文書の配置判定
1. **機能仕様** → `functions/`
   - FUNC-XXX形式で番号管理
   - 個別機能の詳細仕様
2. **バージョン実装計画** → `blueprints/`
   - BP-XXX形式で管理
   - v0.X.Y.Z の全体設計・実装計画
3. **実装コード例** → `code-guides/`
   - CG-XXX形式で管理
   - 具体的な実装パターン・コード例

### 重複防止原則
- **1トピック1ファイル**: 同じ内容の文書は作成しない
- **最新優先**: 古い文書は即座にlegacy/へ移動
- **参照関係**: blueprints → functions → code-guides の階層参照
