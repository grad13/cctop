## 📊 機能概要

All/Uniqueモード切り替え可能な、リアルタイムファイル変更表示UIを提供する機能。

**ユーザー価値**: 直感的なファイル変更監視・効率的な情報表示・操作性の高いUI

## 🎯 機能境界

### ✅ **実行する**
- 画面描画・キーボード入力処理
- データ取得・フォーマット・色分け
- All/Uniqueモード切り替え・リアルタイム更新
- ヘッダー表示・ステータス行表示

### ❌ **実行しない**
- ファイル監視・DB管理・設定管理
- メタデータ収集・ファイルシステム操作

### **表示モード仕様**

#### **Allモード**
- **概要**: 全イベントを時系列順で表示
- **SQLクエリ**: 
```sql
SELECT 
    e.timestamp,
    e.file_name,
    e.directory,
    et.code as event_type,
    e.line_count,
    e.block_count
FROM events e
JOIN event_types et ON e.event_type_id = et.id
ORDER BY e.timestamp DESC 
LIMIT ?
```
- **更新頻度**: 100ms毎のリアルタイム更新
- **スクロール**: 最新イベントが上部に追加

#### **Uniqueモード**
- **概要**: ファイル毎の最新イベントのみ表示
- **SQLクエリ**: 
```sql
SELECT 
    e1.timestamp,
    e1.file_name,
    e1.directory,
    et.code as event_type,
    e1.line_count,
    e1.block_count
FROM events e1
JOIN event_types et ON e1.event_type_id = et.id
INNER JOIN (
    SELECT object_id, MAX(timestamp) as latest
    FROM events 
    GROUP BY object_id
) e2 ON e1.object_id = e2.object_id AND e1.timestamp = e2.latest
ORDER BY e1.timestamp DESC 
LIMIT ?
```
- **更新頻度**: 200ms毎の更新（負荷軽減）

### **キーボード操作仕槕**

| キー | 機能 | 説明 |
|------|------|------|
| `a` | Allモード | 全イベント表示に切り替え |
| `u` | Uniqueモード | ユニークファイル表示に切り替え |
| `q` | 終了 | アプリケーション終了 |

**v0.2.0.0で追加予定の操作**:
- `r`: リフレッシュ
- `↑/↓`: スクロール
- `Space`: 一時停止

## 📈 リアルタイム更新仕様

### **更新戦略**
1. **差分更新**: 前回表示からの変更分のみ描画
2. **バッファリング**: 短時間内の複数更新をまとめて処理
3. **優先制御**: ユーザー操作を描画更新より優先

### **更新フロー**
```
1. DB問い合わせ → 最新データ取得
2. 差分計算 → 前回データとの比較
3. 画面バッファ更新 → 変更部分のみ更新
4. 画面描画 → バッファ内容を画面に反映
5. 待機 → 次回更新まで待機
```

## 🔧 技術仕様

### **依存ライブラリ**
- **chalk**: テキスト色分け（実装済み）
- **cli-table3**: テーブル表示（v0.1.0.0では未使用）

**v0.2.0.0で追加予定**:
- blessed: 高度なUI描画
- moment: 時刻フォーマット

### **設定オプション（config.jsonより）**
- **display.maxEvents**: 最大表示イベント数（デフォルト20）
- **display.refreshRateMs**: 更新間隔（デフォルト100ms）

**CLI引数オーバーライド**:
- `--max-events`: 表示イベント数の一時的上書き
- `--no-color`: 色分け無効化
