# FUNC-011: chokidar-Database統合監視機能

**作成日**: 2025年6月24日  
**作成者**: Architect Agent  
**カテゴリ**: Core Monitoring  
**Phase**: 1 (最優先機能)  
**ステータス**: Active

## 📊 機能概要

chokidarによるファイル変更検出から6項目メタデータ収集・DB記録までの統合処理機能。

**ユーザー価値**: リアルタイムファイル変更検出・詳細メタデータ自動収集・確実な履歴記録

## 🎯 機能境界

### ✅ **実行する**
- chokidarイベント監視・ファイルstat収集
- イベント正規化・6項目メタデータ収集
- activity.dbへのイベント挿入・トランザクション管理
- エラー処理・リトライ機能

### ❌ **実行しない**
- UI表示・設定管理・ユーザー操作処理
- ファイル内容の解析・外部システム連携

## 📋 必要な仕様

### **6項目メタデータ定義**

#### **基本項目（4項目）**
1. **timestamp**: イベント発生時刻（ISO 8601形式）
2. **event_type**: イベント種別（find/create/modify/delete/move/error）
3. **file_path**: 対象ファイルの絶対パス
4. **file_size**: ファイルサイズ（バイト）

#### **詳細項目（2項目）**
5. **line_count**: テキストファイルの行数（バイナリファイルはnull）
6. **inode**: ファイルシステムのinode番号

### **chokidar設定仕様**
```javascript
const chokidarOptions = {
    ignored: [
        '**/node_modules/**',
        '**/.git/**',
        '**/.*',
        '**/activity.db*'
    ],
    persistent: true,
    ignoreInitial: false,
    followSymlinks: false,
    cwd: process.cwd(),
    disableGlobbing: false,
    usePolling: false,
    interval: 100,
    binaryInterval: 300,
    alwaysStat: true,
    depth: undefined,
    awaitWriteFinish: {
        stabilityThreshold: 2000,
        pollInterval: 100
    }
};
```

### **イベント変換テーブル**

| chokidarイベント | cctopイベント | 説明 | readyフラグ |
|-----------------|---------------|------|-------------|
| `add` | `find` | 初期スキャン中のファイル発見 | false |
| `add` | `create` | 監視中の新規ファイル作成 | true |
| `change` | `modify` | ファイル変更検出 | any |
| `unlink` | `delete` | ファイル削除検出 | any |
| `addDir` | `find` | 初期スキャン中のディレクトリ発見 | false |
| `addDir` | `create` | 監視中のディレクトリ作成 | true |
| `unlinkDir` | `delete` | ディレクトリ削除検出 | any |

**注**: `ready`イベントで初期スキャン完了を判定

### **メタデータ収集ロジック**

#### **stat情報取得**
```javascript
const statInfo = {
    size: stats.size,
    mtime: stats.mtime.toISOString(),
    ino: stats.ino
};
```

#### **行数カウント仕様**
- **対象**: テキストファイル（UTF-8, ASCII, Shift_JIS）
- **除外**: バイナリファイル（画像、動画、実行ファイル等）
- **方法**: '\n'文字のカウント
- **上限**: 100万行まで（メモリ保護）

## 📊 データフロー

### **監視フロー**
```
1. chokidar.watch() → ファイルシステム監視開始
2. イベント検出 → add/change/unlink/error
3. stat情報取得 → size/mtime/ino収集
4. 行数カウント → テキストファイルのみ
5. イベント正規化 → cctop形式変換
6. DB挿入 → activity.db events テーブル
```

### **エラーハンドリングフロー**
```
1. エラー検出 → 種別判定（権限/I/O/その他）
2. リトライ判定 → 回復可能性評価
3. リトライ実行 → 最大3回、指数バックオフ
4. エラー記録 → event_type='error'でDB記録
5. 監視継続 → 他ファイルの監視は継続
```
