**作成日**: 2025年6月25日 15:00

**作成者**: Architect Agent  
**ステータス**: Active  
**Version**: 0.2.0.0

## 📊 機能概要

cctopの設定・データベースを現在ディレクトリ（.cctop/）またはユーザーホーム（~/.cctop/）で管理する選択式システム。

**ユーザー価値**: 
- ディレクトリ毎の独立した設定管理
- グローバル設定による複数プロジェクト統一監視
- 設定場所の明示的制御

## 🎯 機能境界

### ✅ **実行する**
- ローカル（.cctop/）をデフォルト使用
- --globalオプションでグローバル（~/.cctop/）使用
- 初回実行時の自動ディレクトリ作成
- 設定場所の明確な表示

### ❌ **実行しない**
- 設定の継承・マージ（シンプルな切り替えのみ）
- 複雑な優先順位システム
- 既存設定の移行・コピー機能

## 📋 必要な仕様

### **基本動作仕様**

#### **1. デフォルト動作**
- カレントディレクトリの`.cctop/`を使用
- 存在しない場合は自動作成してから監視開始
- 作成時に初期設定ファイルも生成

#### **2. オプション指定**

**CLI仕様統合**: 関連するCLIオプションの詳細は **[FUNC-104: CLIインターフェース統合仕様](./FUNC-104-cli-interface-specification.md)** を参照

- `--global`: ホームディレクトリの`~/.cctop/`を使用
- `--local`: 明示的にローカル`.cctop/`を指定（デフォルトと同じ）
- どちらも初回実行時は自動作成

#### **3. ディレクトリ構造**
```
.cctop/
├── config.json      # 設定ファイル
├── activity.db      # イベントデータベース
├── plugins/         # プラグイン用（将来拡張）
├── cache/           # 一時ファイル（オプション）
└── .gitignore       # Git除外設定
```

### **設定ファイル仕様**

設定ファイル（config.json）の詳細構造については、**FUNC-011（階層的設定管理機能）**を参照。

#### **.gitignore内容**
```
# cctop monitoring data
activity.db
activity.db-*
cache/
logs/
```

### **初回実行時のメッセージ**

#### **ローカル設定作成時**
```
✅ Created local configuration in ./.cctop/
📝 Edit ./.cctop/config.json to customize settings
🚀 Starting monitoring...
```

#### **グローバル設定作成時**
```
✅ Created global configuration in ~/.cctop/
📝 Edit ~/.cctop/config.json to customize settings
🚀 Starting monitoring...
```

## 🔧 実装ガイドライン

### **設定パス解決の実装方針**

1. **StoragePathResolver**: 設定ディレクトリパスを解決
   - デフォルト: カレントディレクトリの`.cctop/`
   - `--global`指定時: ホームディレクトリの`~/.cctop/`
   - 存在確認と自動作成処理を統合

2. **ConfigManager**: 設定ファイル管理
   - config.jsonの読み込み・検証
   - 初回実行時のデフォルト設定作成
   - データベースパスの相対パス解決

3. **CLIインターフェース**: コマンドライン引数処理
   - `--global/--local`: モード切り替え
   - 初回実行を検出して自動初期化
   - 作成場所の明確な表示

## 💡 使用シナリオ

### **基本的な使用フロー**
```bash
cd my-project
cctop                 # 初回: .cctop/自動作成して監視開始
cctop                 # 2回目以降: 既存設定で監視開始
```

### **グローバル設定の使用**
```bash
# ホームディレクトリの共通設定を使用
cctop --global        # 初回: ~/.cctop/自動作成
cd any-directory
cctop --global        # どのディレクトリでも同じ設定
```

### **明示的なローカル指定**
```bash
# デフォルトと同じだが、意図を明確にする場合
cctop --local         # .cctop/を使用
```

## 🎯 成功指標

1. **ゼロ設定起動**: 初回実行時も即座に使用開始可能
2. **明示的制御**: --globalオプションで設定場所を選択
3. **シンプル**: 設定の継承なし、明快な動作ルール
